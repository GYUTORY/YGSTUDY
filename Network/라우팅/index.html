    default (0.0.0.0/0) л§Өм№ӯ /0
    10.0.0.0/8 л§Өм№ӯ /8
    10.10.0.0/16 л§Өм№ӯ /16

    4к°ң лӢӨ л§Өм№ӯлҗҳм§Җл§Ң, /24к°Җ к°ҖмһҘ кёёлӢӨ. к·ёлһҳм„ң 10.0.0.250мңјлЎң ліҙлӮёлӢӨ.

    мқҙкұё лӘЁлҘҙл©ҙ "л¶„лӘ… лқјмҡ°нҢ… н…Ңмқҙлё”м—җ кІҪлЎңлҘј л„Јм—ҲлҠ”лҚ° мҷң лӢӨлҘё кёёлЎң к°Җм§Җ?"лқјлҠ” мғҒнҷ©мқ„ мқҙн•ҙн•  мҲҳ м—ҶлӢӨ. static routeлҘј м¶”к°Җн–ҲлҠ”лҚ° лҸҷмһ‘н•ҳм§Җ м•ҠлҠ” кІғмІҳлҹј ліҙмқҙл©ҙ, лҚ” кө¬мІҙм Ғмқё кІҪлЎңк°Җ мқҙлҜё мһҲлҠ”м§Җ нҷ•мқён•ҙм•ј н•ңлӢӨ.

    2. Static Routing vs Dynamic Routing¶

    Static RouteлЎң м¶©л¶„н•ң кІҪмҡ°¶

    м„ңлІ„ 2~3лҢҖ, л„ӨнҠёмӣҢнҒ¬ кІҪлЎңк°Җ н•ҳлӮҳлҝҗмқё нҷҳкІҪм—җм„ңлҠ” static routeл©ҙ лҗңлӢӨ. лҢҖл¶Җл¶„мқҳ лӢЁмқј м„ңлёҢл„· нҷҳкІҪмқҙ м—¬кё°м—җ н•ҙлӢ№н•ңлӢӨ.

    # мӮ¬лӮҙ л„ӨнҠёмӣҢнҒ¬(192.168.10.0/24)лЎң к°ҖлҠ” кІҪлЎң м¶”к°Җ
ip route add 192.168.10.0/24 via 10.0.0.1

# кё°ліё кІҢмқҙнҠёмӣЁмқҙ м„Өм •
ip route add default via 10.0.0.1

# мһ¬л¶ҖнҢ… нӣ„м—җлҸ„ мң м§Җн•ҳл Өл©ҙ /etc/netplan/ лҳҗлҠ” /etc/sysconfig/network-scripts/м—җ м„Өм •

    static routeмқҳ л¬ём ңлҠ” кІҪлЎңк°Җ мЈҪм–ҙлҸ„ лқјмҡ°нҢ… н…Ңмқҙлё”мқҙ л°”лҖҢм§Җ м•ҠлҠ”лӢӨлҠ” кІғмқҙлӢӨ. viaлЎң м§Җм •н•ң кІҢмқҙнҠёмӣЁмқҙк°Җ лӢӨмҡҙлҗҳл©ҙ нҢЁнӮ·мқҖ к·ёлғҘ лё”лһҷнҷҖм—җ л№ м§„лӢӨ.

    Dynamic Routingмқҙ н•„мҡ”н•ҙм§ҖлҠ” мӢңм җ¶

    мқҙлҹ° мғҒнҷ©мқҙ мҳӨл©ҙ OSPF к°ҷмқҖ dynamic routingмқ„ кі л Өн•ҙм•ј н•ңлӢӨ:

    • м„ңлІ„к°Җ м—¬лҹ¬ л„ӨнҠёмӣҢнҒ¬ м„ёк·ёлЁјнҠём—җ кұёміҗ мһҲкі , кІҪлЎңк°Җ 2к°ң мқҙмғҒмқё кІҪмҡ°
    • лқјмҡ°н„° н•ҳлӮҳк°Җ мЈҪм—Ҳмқ„ л•Ң мһҗлҸҷмңјлЎң мҡ°нҡҢ кІҪлЎңлҘј мһЎм•„м•ј н•ҳлҠ” кІҪмҡ°
    • л„ӨнҠёмӣҢнҒ¬ нҶ нҸҙлЎңм§Җк°Җ мһҗмЈј л°”лҖҢлҠ” кІҪмҡ°

    OSPFлҠ” к°Ғ лқјмҡ°н„°к°Җ мһҗкё° мЈјліҖ л„ӨнҠёмӣҢнҒ¬ мғҒнғңлҘј кҙ‘кі н•ҳкі , лӘЁл“  лқјмҡ°н„°к°Җ м „мІҙ л„ӨнҠёмӣҢнҒ¬ м§ҖлҸ„лҘј к°Җм§Җкі  мөңлӢЁ кІҪлЎңлҘј кі„мӮ°н•ңлӢӨ. лқјмҡ°н„° н•ҳлӮҳк°Җ мЈҪмңјл©ҙ лӘҮ мҙҲ м•Ҳм—җ мғҲ кІҪлЎңлҘј кі„мӮ°н•ҙм„ң н…Ңмқҙлё”мқ„ м—…лҚ°мқҙнҠён•ңлӢӨ.

    # Linuxм—җм„ң OSPFлҘј м“°л Өл©ҙ FRRouting(frr)мқ„ м„Өм№ҳн•ңлӢӨ
# /etc/frr/ospfd.conf

router ospf
  ospf router-id 10.0.0.15
  network 10.0.0.0/24 area 0
  network 192.168.1.0/24 area 0

    мӢӨл¬ҙм—җм„ң м„ңлІ„ м—”м§ҖлӢҲм–ҙк°Җ OSPFлҘј м§Ғм ‘ м„Өм •н•  мқјмқҖ л§Һм§Җ м•ҠлӢӨ. ліҙнҶөмқҖ л„ӨнҠёмӣҢнҒ¬ нҢҖмқҙ мҠӨмң„м№ҳ/лқјмҡ°н„°м—җм„ң кҙҖлҰ¬н•ңлӢӨ. н•ҳм§Җл§Ң Linux м„ңлІ„к°Җ лқјмҡ°н„° м—ӯн• мқ„ н•ҙм•ј н•ҳлҠ” кІҪмҡ°(VPN кІҢмқҙнҠёмӣЁмқҙ, л©ҖнӢ°нҷҲ м„ңлІ„ л“ұ)м—җлҠ” FRRoutingмңјлЎң OSPFлҘј лҸҢлҰ¬кІҢ лҗңлӢӨ.

    BGPлҠ” лӢӨлҘё м°Ёмӣҗмқҳ мқҙм•јкё°лӢӨ. ISP к°„ лқјмҡ°нҢ…, л©ҖнӢ° нҒҙлқјмҡ°л“ң м—°кІ°, CDNм—җм„ң мӮ¬мҡ©н•ңлӢӨ. мқјл°ҳм Ғмқё м„ңл№„мҠӨ мҡҙмҳҒм—җм„ң BGPлҘј м§Ғм ‘ лӢӨлЈ° мқјмқҖ кұ°мқҳ м—ҶлӢӨ. лӢӨл§Ң BGP hijacking к°ҷмқҖ мӮ¬кі лҘј мқҙн•ҙн•ҳл Өл©ҙ кё°ліё к°ңл…җмқҖ м•Ңм•„л‘¬м•ј н•ҳлҠ”лҚ°, мқҙкұҙ нҠёлҹ¬лё”мҠҲнҢ… м„№м…ҳм—җм„ң лӢӨлЈ¬лӢӨ.

    3. Linux Policy Routing¶

    л¬ём ң мғҒнҷ©: NICк°Җ 2к°ңмқё м„ңлІ„¶

    м„ңлІ„м—җ NICк°Җ 2к°ң кҪӮнҳҖ мһҲлӢӨкі  н•ҳмһҗ.

    • eth0: 10.0.0.15/24 (кІҢмқҙнҠёмӣЁмқҙ 10.0.0.1) вҖ” мӮ¬лӮҙ л„ӨнҠёмӣҢнҒ¬
    • eth1: 203.0.113.15/24 (кІҢмқҙнҠёмӣЁмқҙ 203.0.113.1) вҖ” мҷёл¶Җ м„ңл№„мҠӨмҡ©

    кё°ліё лқјмҡ°нҢ… н…Ңмқҙлё”л§ҢмңјлЎңлҠ” л¬ём ңк°Җ мғқкёҙлӢӨ. мҷёл¶Җм—җм„ң 203.0.113.15лЎң л“Өм–ҙмҳЁ мҡ”мІӯмқҳ мқ‘лӢөмқҙ default routeлҘј нғҖкі  eth0(10.0.0.1)мңјлЎң лӮҳк°ҖлІ„лҰҙ мҲҳ мһҲлӢӨ. л“Өм–ҙмҳЁ кіікіј лӢӨлҘё кіімңјлЎң лӮҳк°Җл©ҙ л°©нҷ”лІҪмқҙ м°ЁлӢЁн•ҳкұ°лӮҳ, нҒҙлқјмқҙм–ёнҠё мһ…мһҘм—җм„ң мқ‘лӢөмқҳ м¶ңл°ңм§Җ IPк°Җ лӢ¬лқјм ём„ң м—°кІ°мқҙ лҒҠкёҙлӢӨ.

    н•ҙкІ°: source-based routing¶

    # 1. лі„лҸ„ лқјмҡ°нҢ… н…Ңмқҙлё” м •мқҳ
echo "100 external" >> /etc/iproute2/rt_tables

# 2. external н…Ңмқҙлё”м—җ кІҪлЎң м¶”к°Җ
ip route add 203.0.113.0/24 dev eth1 table external
ip route add default via 203.0.113.1 table external

# 3. м¶ңл°ңм§Җк°Җ 203.0.113.15мқё нҢЁнӮ·мқҖ external н…Ңмқҙлё”мқ„ мӮ¬мҡ©н•ҳлҸ„лЎқ лЈ° м¶”к°Җ

    мқҙл ҮкІҢ н•ҳл©ҙ:

    • eth1(203.0.113.15)лЎң л“Өм–ҙмҳЁ мҡ”мІӯм—җ лҢҖн•ң мқ‘лӢө вҶ’ м¶ңл°ңм§Җк°Җ 203.0.113.15 вҶ’ external н…Ңмқҙлё” м°ёмЎ° вҶ’ eth1/203.0.113.1лЎң лӮҳк°җ
    • eth0(10.0.0.15)лЎң л“Өм–ҙмҳЁ мҡ”мІӯм—җ лҢҖн•ң мқ‘лӢө вҶ’ кё°ліё main н…Ңмқҙлё” м°ёмЎ° вҶ’ eth0/10.0.0.1лЎң лӮҳк°җ

    нҳ„мһ¬ лЈ° лӘ©лЎқ нҷ•мқё:

    $ ip rule show

100:    from 203.0.113.15 lookup external
32766:  from all lookup main
32767:  from all lookup default

    priority мҲ«мһҗк°Җ мһ‘мқ„мҲҳлЎқ лЁјм Җ нҸүк°ҖлҗңлӢӨ. local н…Ңмқҙлё”(priority 0)мқҖ н•ӯмғҒ мІ« лІҲм§ёлЎң, лЎңм»¬ мЈјмҶҢмҷҖ лёҢлЎңл“ңмәҗмҠӨнҠё мЈјмҶҢлҘј мІҳлҰ¬н•ңлӢӨ.

    мЈјмқҳн•  м җ: ip routeмҷҖ ip ruleлЎң м¶”к°Җн•ң м„Өм •мқҖ мһ¬л¶ҖнҢ…н•ҳл©ҙ мӮ¬лқјм§„лӢӨ. мҳҒкө¬ м Ғмҡ©н•ҳл Өл©ҙ netplan(Ubuntu) лҳҗлҠ” NetworkManager dispatcher мҠӨнҒ¬лҰҪнҠём—җ л„Јм–ҙм•ј н•ңлӢӨ.

    4. Docker / Kubernetes лқјмҡ°нҢ…¶

    Docker bridge л„ӨнҠёмӣҢнҒ¬¶

    DockerлҘј м„Өм№ҳн•ҳл©ҙ docker0мқҙлқјлҠ” bridge мқён„°нҺҳмқҙмҠӨк°Җ мғқкёҙлӢӨ. м»Ён…Ңмқҙл„ҲлҘј мӢӨн–үн•ҳл©ҙ к°Ғ м»Ён…Ңмқҙл„Ҳм—җ veth нҺҳм–ҙк°Җ мғқкё°кі , н•ңмӘҪмқҖ м»Ён…Ңмқҙл„Ҳ м•Ҳмқҳ eth0, лӢӨлҘё мӘҪмқҖ нҳёмҠӨнҠёмқҳ docker0 лёҢлҰҝм§Җм—җ л¶ҷлҠ”лӢӨ.

    нҳёмҠӨнҠё
в”ңв”Җв”Җ eth0 (10.0.0.15)
в”ңв”Җв”Җ docker0 (172.17.0.1)
в”Ӯ   в”ңв”Җв”Җ veth1 вҶҗвҶ’ container1 eth0 (172.17.0.2)
в”Ӯ   в”ңв”Җв”Җ veth2 вҶҗвҶ’ container2 eth0 (172.17.0.3)
в”Ӯ   в””в”Җв”Җ veth3 вҶҗвҶ’ container3 eth0 (172.17.0.4)

    к°ҷмқҖ bridgeм—җ л¶ҷмқҖ м»Ён…Ңмқҙл„ҲлҒјлҰ¬ нҶөмӢ н•  л•ҢлҠ” L2(MAC мЈјмҶҢ) мҲҳмӨҖм—җм„ң мІҳлҰ¬лҗңлӢӨ. docker0мқҙ мқјл°ҳ л„ӨнҠёмӣҢнҒ¬ мҠӨмң„м№ҳмІҳлҹј лҸҷмһ‘н•ңлӢӨ. лқјмҡ°нҢ…мқҙ м•„лӢҲлқј лёҢлҰҝм§•мқҙлӢӨ.

    м»Ён…Ңмқҙл„Ҳм—җм„ң мҷёл¶Җ(мқён„°л„·)лЎң лӮҳк°ҖлҠ” нҢЁнӮ·мқҖ лӢӨлҘё кіјм •мқ„ кұ°м№ңлӢӨ:

    1. м»Ён…Ңмқҙл„Ҳмқҳ default routeк°Җ 172.17.0.1(docker0)мқ„ к°ҖлҰ¬нӮЁлӢӨ
    2. нҳёмҠӨнҠёмқҳ iptables NAT к·ңм№ҷмқҙ м¶ңл°ңм§Җ IPлҘј нҳёмҠӨнҠё IPлЎң ліҖнҷҳн•ңлӢӨ (MASQUERADE)
    3. нҳёмҠӨнҠёмқҳ default routeлҘј нғҖкі  мҷёл¶ҖлЎң лӮҳк°„лӢӨ
    # нҳёмҠӨнҠём—җм„ң нҷ•мқё

Chain POSTROUTING (policy ACCEPT)
 pkts bytes target     prot opt in     out     source         destination
  142  9568 MASQUERADE  all  --  *      !docker0  172.17.0.0/16  0.0.0.0/0

    !docker0 вҖ” docker0мңјлЎң лӮҳк°ҖлҠ” кұҙ м ңмҷё(к°ҷмқҖ bridge лӮҙ нҶөмӢ ). к·ё мҷё лӘЁл“  нҢЁнӮ·мқҳ м¶ңл°ңм§ҖлҘј MASQUERADE(нҳёмҠӨнҠё IPлЎң ліҖнҷҳ)н•ңлӢӨ.

    Kubernetes Service лқјмҡ°нҢ… (iptables лӘЁл“ң)¶

    Kubernetesм—җм„ң ServiceлҘј л§Ңл“Өл©ҙ ClusterIP(к°ҖмғҒ IP)к°Җ н• лӢ№лҗңлӢӨ. мқҙ IPлҠ” м–ҙл–Ө л„ӨнҠёмӣҢнҒ¬ мқён„°нҺҳмқҙмҠӨм—җлҸ„ л°”мқёл”©лҗҳм§Җ м•ҠлҠ”лӢӨ вҖ” мҲңмҲҳн•ҳкІҢ iptables к·ңм№ҷмңјлЎңл§Ң мЎҙмһ¬н•ңлӢӨ.

    Podк°Җ Service IPлЎң нҢЁнӮ·мқ„ ліҙлӮҙл©ҙ:

    л¬ём ңлҠ” Pod мҲҳк°Җ л§Һм•„м§Җл©ҙ iptables к·ңм№ҷ мҲҳк°Җ O(n)мңјлЎң мҰқк°Җн•ңлӢӨлҠ” кІғмқҙлӢӨ. Service 1000к°ң, Pod 10000к°ң мҲҳмӨҖмқҙ лҗҳл©ҙ iptables к·ңм№ҷ м—…лҚ°мқҙнҠём—җ мҲҳмӢӯ мҙҲк°Җ кұёлҰҙ мҲҳ мһҲлӢӨ. мқҙлһҳм„ң лҢҖк·ңлӘЁ нҒҙлҹ¬мҠӨн„°м—җм„ңлҠ” IPVS лӘЁл“ңлҘј м“°кұ°лӮҳ Cilium к°ҷмқҖ eBPF кё°л°ҳ л„ӨнҠёмӣҢнӮ№мңјлЎң көҗмІҙн•ңлӢӨ.

    5. AWS VPC лқјмҡ°нҢ…¶

    public м„ңлёҢл„·кіј private м„ңлёҢл„·мқҳ м°Ёмқҙ¶

    publicкіј private м„ңлёҢл„·мқҳ м°ЁмқҙлҠ” л”ұ н•ң мӨ„мқҙлӢӨ:

    • public м„ңлёҢл„·: 0.0.0.0/0 вҶ’ Internet Gateway
    • private м„ңлёҢл„·: 0.0.0.0/0 вҶ’ NAT Gateway

    Internet Gateway(IGW)лҠ” м–‘л°©н–ҘмқҙлӢӨ. мқёмҠӨн„ҙмҠӨм—җ Public IPк°Җ мһҲмңјл©ҙ мҷёл¶Җм—җм„ң м§Ғм ‘ м ‘к·јн•  мҲҳ мһҲлӢӨ. NAT GatewayлҠ” м•„мӣғл°”мҡҙл“ңл§Ң н—Ҳмҡ©н•ңлӢӨ. private м„ңлёҢл„·мқҳ мқёмҠӨн„ҙмҠӨк°Җ нҢЁнӮӨм§ҖлҘј лӢӨмҡҙлЎңл“ңн•ҳкұ°лӮҳ мҷёл¶Җ APIлҘј нҳём¶ңн•  мҲҳ мһҲм§Җл§Ң, мҷёл¶Җм—җм„ң лЁјм Җ м ‘к·јн•  мҲҳлҠ” м—ҶлӢӨ.

    мқён„°л„· вҶ’ IGW вҶ’ public м„ңлёҢл„· (10.0.1.0/24)
                    вҶ“ (NAT Gatewayк°Җ м—¬кё° мң„м№ҳ)

    NAT GatewayлҠ” л°ҳл“ңмӢң public м„ңлёҢл„·м—җ л§Ңл“Өм–ҙм•ј н•ңлӢӨ. NAT Gateway мһҗмІҙк°Җ мқён„°л„·м—җ м ‘к·јн•ҙм•ј н•ҳлӢҲк№Ң. private м„ңлёҢл„·м—җ NAT GatewayлҘј л§Ңл“ңлҠ” мӢӨмҲҳлҘј к°ҖлҒ” ліҙлҠ”лҚ°, лқјмҡ°нҢ…мқҙ мҲңнҷҳ м°ёмЎ°м—җ л№ м ём„ң мҷёл¶Җ нҶөмӢ мқҙ м „нҳҖ м•Ҳ лҗңлӢӨ.

    • * * * вҖ” н•ҙлӢ№ нҷүмқҳ лқјмҡ°н„°к°Җ ICMP Time ExceededлҘј ліҙлӮҙм§Җ м•ҠлҠ” кІғмқҙлӢӨ. л°©нҷ”лІҪм—җм„ң л§үм•ҳкұ°лӮҳ, лқјмҡ°н„° м„Өм •мңјлЎң л№„нҷңм„ұнҷ”н•ң кІҪмҡ°. к·ё л’Ө нҷүмқҙ м •мғҒмқҙл©ҙ л¬ём ңк°Җ м•„лӢҲлӢӨ.
    • нҠ№м • нҷүл¶Җн„° кі„мҶҚ * * * вҖ” н•ҙлӢ№ нҷүм—җм„ң нҢЁнӮ·мқҙ л“ңлЎӯлҗҳкұ°лӮҳ, к·ё мқҙнӣ„ кІҪлЎңм—җ л¬ём ңк°Җ мһҲлӢӨ.
    • RTTк°Җ к°‘мһҗкё° нҒ¬кІҢ лӣ°лҠ” нҷү вҖ” н•ҙлӢ№ кө¬к°„мқҳ л§ҒнҒ¬к°Җ лҠҗлҰ¬кұ°лӮҳ, м§ҖлҰ¬м ҒмңјлЎң лЁј кіімңјлЎң л„ҳм–ҙк°ҖлҠ” м§Җм җмқҙлӢӨ. лҢҖм–‘ нҡЎлӢЁ кө¬к°„м—җм„ң 50~100ms м •лҸ„ лӣ°лҠ” кұҙ м •мғҒмқҙлӢӨ.

    -n мҳөм…ҳмқҖ DNS м—ӯм§ҲмқҳлҘј м•Ҳ н•ҳкІ лӢӨлҠ” лң»мқҙлӢӨ. м—ӯм§Ҳмқҳк°Җ лҠҗлҰ¬л©ҙ traceroute мһҗмІҙк°Җ мҳӨлһҳ кұёл Өм„ң, ліҙнҶө -nмқ„ л¶ҷм—¬м„ң мӢӨн–үн•ңлӢӨ.

    л№„лҢҖм№ӯ лқјмҡ°нҢ… (Asymmetric Routing)¶

    мҡ”мІӯмқҙ A вҶ’ B вҶ’ C вҶ’ D кІҪлЎңлЎң к°”лҠ”лҚ°, мқ‘лӢөмқҙ D вҶ’ E вҶ’ A кІҪлЎңлЎң лҸҢм•„мҳӨлҠ” кІҪмҡ°лӢӨ. кІҪлЎң мһҗмІҙлҠ” л¬ём ңк°Җ м•„лӢҢлҚ°, stateful л°©нҷ”лІҪмқҙ мһҲмңјл©ҙ л¬ём ңк°Җ лҗңлӢӨ.

    stateful л°©нҷ”лІҪмқҖ мҡ”мІӯ нҢЁнӮ·мқ„ ліҙкі  "мқҙ м—°кІ°мқҙ м—ҙл ёлӢӨ"кі  кё°лЎқн•ңлӢӨ. мқ‘лӢөмқҙ к°ҷмқҖ кІҪлЎңлЎң лҸҢм•„мҷҖм•ј кё°лЎқкіј л§Өм№ӯн•ҙм„ң нҶөкіјмӢңнӮӨлҠ”лҚ°, лӢӨлҘё кІҪлЎңлЎң лҸҢм•„мҳӨл©ҙ л°©нҷ”лІҪмқҙ "мқҙкұҙ мғҲлЎңмҡҙ м—°кІ°мқёлҚ° SYN м—Ҷмқҙ ACKк°Җ мҷ”л„Ө"лқјкі  нҢҗлӢЁн•ҳкі  л“ңлЎӯн•ңлӢӨ.

    мҰқмғҒ: - pingмқҖ лҗҳлҠ”лҚ° TCP м—°кІ°мқҙ м•Ҳ лҗңлӢӨ - SYNмқҖ ліҙлӮҙм§ҖлҠ”лҚ° SYN-ACKк°Җ м•Ҳ лҸҢм•„мҳЁлӢӨ - к°„н—җм ҒмңјлЎң м—°кІ°мқҙ лҒҠкёҙлӢӨ (кІҪлЎңк°Җ л°”лҖ” л•Ңл§Ң)

    нҷ•мқё л°©лІ•: 

    # м–‘мӘҪм—җм„ң tcpdumpлЎң нҢЁнӮ· мәЎмІҳ
tcpdump -i eth0 host 10.0.0.15 -nn

# м–‘мӘҪм—җм„ң traceroute вҖ” кІҪлЎңк°Җ лӢӨлҘҙл©ҙ л№„лҢҖм№ӯ
traceroute -n 10.0.0.15     # A вҶ’ B
traceroute -n 10.0.0.100    # B вҶ’ A (кІҪлЎңк°Җ лӢӨлҰ„)

    Linux м„ңлІ„м—җм„ң л№„лҢҖм№ӯ лқјмҡ°нҢ…мқҙ л¬ём ңк°Җ лҗҳл©ҙ, rp_filter(reverse path filtering) м„Өм •мқ„ нҷ•мқён•ңлӢӨ.

    # нҳ„мһ¬ м„Өм • нҷ•мқё
sysctl net.ipv4.conf.all.rp_filter
sysctl net.ipv4.conf.eth0.rp_filter

# 0: кІҖмӮ¬ м•Ҳ н•Ё, 1: strict (л“Өм–ҙмҳЁ мқён„°нҺҳмқҙмҠӨлЎң лҸҢм•„к°Ҳ мҲҳ мһҲлҠ”м§Җ нҷ•мқё), 2: loose (м–ҙл–Ө кІҪлЎңл“  мЎҙмһ¬н•ҳл©ҙ OK)
# л№„лҢҖм№ӯ лқјмҡ°нҢ…мқҙ м •мғҒмқё нҷҳкІҪм—җм„ңлҠ” 2(loose)лЎң м„Өм •
sysctl -w net.ipv4.conf.eth0.rp_filter=2

    MTU л¬ём ңлЎң нҢЁнӮ·мқҙ мӮ¬лқјм§ҖлҠ” мјҖмқҙмҠӨ¶

    TCP м—°кІ°мқҖ лҗҳлҠ”лҚ° лҚ°мқҙн„°лҘј ліҙлӮҙл©ҙ л©Ҳм¶”лҠ” мҰқмғҒмқҙ мһҲлӢӨ. мһ‘мқҖ нҢЁнӮ·(SYN, ACK)мқҖ мһҳ к°ҖлҠ”лҚ°, мӢӨм ң лҚ°мқҙн„°лҘј лӢҙмқҖ нҒ° нҢЁнӮ·мқҙ мӨ‘к°„м—җ мӮ¬лқјм§ҖлҠ” кІҪмҡ°лӢӨ.

    мӣҗмқё: кІҪлЎң мӨ‘к°„м—җ MTUк°Җ мһ‘мқҖ кө¬к°„мқҙ мһҲлҠ”лҚ°, DF(Don't Fragment) л№„нҠёк°Җ м„Өм •лҗҳм–ҙ мһҲкі , ICMP Fragmentation Needed л©”мӢңм§Җк°Җ л°©нҷ”лІҪм—җ мқҳн•ҙ м°ЁлӢЁлҗЁ.

    мқҙкұё Path MTU Discovery Black Holeмқҙлқјкі  н•ңлӢӨ.

    м„ңлІ„ (MTU 1500) вҶ’ лқјмҡ°н„° (MTU 1500) вҶ’ н„°л„җ/VPN (MTU 1400) вҶ’ лӘ©м Ғм§Җ

1. м„ңлІ„к°Җ 1500л°”мқҙнҠё нҢЁнӮ·мқ„ ліҙлғ„ (DF л№„нҠё м„Өм •)
2. н„°л„җ кө¬к°„м—җм„ң 1400ліҙлӢӨ нҒ¬лӢҲк№Ң м „лӢ¬н•  мҲҳ м—ҶмқҢ
3. н„°л„җ мһҘл№„к°Җ ICMP "Fragmentation Needed" л©”мӢңм§ҖлҘј ліҙлғ„
4. к·јлҚ° мӨ‘к°„ л°©нҷ”лІҪмқҙ ICMPлҘј м „л¶Җ л§үм•„л‘ 
5. м„ңлІ„лҠ” ICMPлҘј лӘ» л°ӣм•„м„ң MTUлҘј мӨ„мқј мҲҳ м—Ҷкі , кі„мҶҚ 1500мңјлЎң ліҙлғ„
6. нҢЁнӮ·мқҙ кі„мҶҚ л“ңлЎӯлҗЁ

    мҰқмғҒ нҷ•мқё:

    # pingмңјлЎң MTU нғҗмғү
ping -M do -s 1472 лӘ©м Ғм§ҖIP    # 1472 + 28(IP+ICMP н—ӨлҚ”) = 1500
ping -M do -s 1372 лӘ©м Ғм§ҖIP    # 1372 + 28 = 1400 вҖ” мқҙкІҢ лҗҳл©ҙ мӨ‘к°„м—җ MTU 1400 кө¬к°„мқҙ мһҲлҠ” кІғ

# tcpdumpлЎң ліҙл©ҙ нҒ° нҢЁнӮ·л§Ң мқ‘лӢөмқҙ м—ҶмқҢ
tcpdump -i eth0 host лӘ©м Ғм§ҖIP -nn

    н•ҙкІ° л°©лІ•:

    # л°©лІ• 1: мқён„°нҺҳмқҙмҠӨ MTUлҘј лӮ®м¶Ө
ip link set eth0 mtu 1400

# л°©лІ• 2: TCP MSS clamping вҖ” iptablesлЎң TCP н•ёл“ңм…°мқҙнҒ¬ мӢң MSS к°’мқ„ мЎ°м •
iptables -t mangle -A FORWARD -p tcp --tcp-flags SYN,RST SYN \
  -j TCPMSS --clamp-mss-to-pmtu

# л°©лІ• 3: MSSлҘј м§Ғм ‘ м§Җм •
iptables -t mangle -A FORWARD -p tcp --tcp-flags SYN,RST SYN \
  -j TCPMSS --set-mss 1360

    VPN, GRE н„°л„җ, VXLANмқ„ мӮ¬мҡ©н•ҳлҠ” нҷҳкІҪм—җм„ң мқҙ л¬ём ңк°Җ мһҗмЈј л°ңмғқн•ңлӢӨ. AWSм—җм„ң VPC к°„ нҶөмӢ н•  л•ҢлҸ„ м җліҙ н”„л Ҳмһ„мқ„ м“°лӢӨк°Җ VPN м—°кІ°лЎң л„ҳм–ҙк°ҖлҠ” кө¬к°„м—җм„ң л§ҢлӮ  мҲҳ мһҲлӢӨ.

    BGP Hijacking¶

    BGPлҠ” AS(Autonomous System) к°„ лқјмҡ°нҢ… н”„лЎңнҶ мҪңмқёлҚ°, к·јліём ҒмңјлЎң мӢ лў° кё°л°ҳмқҙлӢӨ. м–ҙл–Ө ASк°Җ "лӮҳлҠ” 1.1.1.0/24лҘј к°Җм§Җкі  мһҲлӢӨ"кі  кҙ‘кі н•ҳл©ҙ, мқҙмӣғ ASл“ӨмқҖ к·ёкұё к·ёлҢҖлЎң лҜҝкі  лқјмҡ°нҢ… н…Ңмқҙлё”м—җ л°ҳмҳҒн•ңлӢӨ.

    BGP hijackingмқҖ мӢӨм ң мҶҢмң мһҗк°Җ м•„лӢҢ ASк°Җ нҠ№м • IP лҢҖм—ӯмқ„ мһҗкё° кІғмқҙлқјкі  кҙ‘кі н•ҳлҠ” кІғмқҙлӢӨ. longest prefix match л•Ңл¬ём—җ лҚ” кө¬мІҙм Ғмқё кІҪлЎңлҘј кҙ‘кі н•ҳл©ҙ мүҪкІҢ нҠёлһҳн”Ҫмқ„ к°ҖлЎңмұҢ мҲҳ мһҲлӢӨ.

    мӢӨм ң мӮ¬лЎҖ: - 2018л…„ Amazon Route 53 BGP hijack вҖ” кіөкІ©мһҗк°Җ Amazonмқҙ мҶҢмң н•ң IP лҢҖм—ӯмқ„ 2мӢңк°„ лҸҷм•Ҳ к°ҖлЎңмұ„м„ң MyEtherWallet мӮ¬мҡ©мһҗмқҳ DNS м§ҲмқҳлҘј мһҗкё° м„ңлІ„лЎң лҸҢл ёлӢӨ. кІ°кіјм ҒмңјлЎң м•”нҳёнҷ”нҸҗ м•Ҫ 15л§Ң лӢ¬лҹ¬лҘј нғҲм·Ён–ҲлӢӨ. - 2008л…„ нҢҢнӮӨмҠӨнғ„ н…”л ҲмҪӨ YouTube м°ЁлӢЁ вҖ” нҢҢнӮӨмҠӨнғ„ м •л¶Җмқҳ YouTube м°ЁлӢЁ лӘ…л №мқ„ мӢӨн–үн•ҳл©ҙм„ң, ISPк°Җ YouTube IP лҢҖм—ӯм—җ лҢҖн•ң лҚ” кө¬мІҙм Ғмқё кІҪлЎңлҘј BGPлЎң кҙ‘кі н•ҙлІ„л ёлӢӨ. мқҙ кІҪлЎңк°Җ м „ м„ёкі„лЎң нҚјм ём„ң 2мӢңк°„ лҸҷм•Ҳ м „ м„ёкі„м—җм„ң YouTube м ‘мҶҚмқҙ л¶Ҳк°ҖлҠҘн–ҲлӢӨ.

    м„ңл№„мҠӨ мҡҙмҳҒмһҗ мһ…мһҘм—җм„ң н•  мҲҳ мһҲлҠ” л°©м–ҙ: - RPKI(Resource Public Key Infrastructure) вҖ” IP лҢҖм—ӯмқҳ мҶҢмң к¶Ңмқ„ м•”нҳён•ҷм ҒмңјлЎң кІҖмҰқн•ңлӢӨ. ROA(Route Origin Authorization)лҘј л°ңкёүн•ҙм„ң "мқҙ IP лҢҖм—ӯмқҖ AS XXXXXл§Ң кҙ‘кі н•  мҲҳ мһҲлӢӨ"кі  м„ м–ён•ңлӢӨ. AWS, GCP к°ҷмқҖ нҒҙлқјмҡ°л“ң н”„лЎңл°”мқҙлҚ”л“ӨмқҖ мқҙлҜё RPKIлҘј м§Җмӣҗн•ңлӢӨ. - BGP лӘЁлӢҲн„°л§Ғ вҖ” мһҗкё° IP лҢҖм—ӯмқҙ лӢӨлҘё ASм—җм„ң кҙ‘кі лҗҳлҠ”м§Җ к°җмӢңн•ңлӢӨ. RIPE RIS, BGPStream к°ҷмқҖ м„ңл№„мҠӨлЎң нҷ•мқён•  мҲҳ мһҲлӢӨ.
