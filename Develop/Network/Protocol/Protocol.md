# 네트워크 프로토콜(Protocol)

## 정의

프로토콜(Protocol)은 컴퓨터 네트워크에서 서로 다른 시스템 간에 데이터를 주고받기 위해 정해진 통신 규칙의 집합이다.

**실무 팁:**
프로토콜은 네트워크 통신의 기본 규칙이다. HTTP, TCP, UDP 등 다양한 프로토콜이 있으며, 각각의 용도가 다르다.

### 프로토콜이 필요한 이유

**1. 상호 운용성 (Interoperability)**
- 다양한 기기 간 통신 가능
- 플랫폼 독립성
- 표준화된 인터페이스

**2. 데이터 무결성 (Data Integrity)**
- 전송 오류 감지
- 순서 보장
- 중복 제거

**3. 보안과 안정성**
- 암호화
- 인증
- 오류 복구

## 프로토콜의 주요 요소

### 1. 구문(Syntax) - 데이터 형식

데이터를 어떤 형태로 만들지 정하는 규칙입니다.

**HTTP의 구문 예시**
```
GET /api/users HTTP/1.1
Host: example.com
Content-Type: application/json
Authorization: Bearer token123
```

- 요청 라인: `GET /api/users HTTP/1.1` (메서드, 경로, 버전)
- 헤더: `Host`, `Content-Type` 등
- 본문: 실제 데이터

### 2. 의미론(Semantics) - 데이터 해석

데이터가 어떤 뜻을 가지고 있는지 정의합니다.

**HTTP 상태 코드**
- 200 OK: 요청 성공
- 404 Not Found: 리소스 없음
- 500 Internal Server Error: 서버 오류

**TCP 플래그**
- SYN: 연결 시작 요청
- ACK: 데이터 수신 확인
- FIN: 연결 종료 요청
- RST: 연결 강제 중단

### 3. 타이밍(Timing) - 전송 시점

데이터를 언제, 얼마나 빠르게 보낼지 결정합니다.

**TCP 3-way Handshake**
1. SYN: 클라이언트가 서버에 연결 요청
2. SYN-ACK: 서버가 연결 수락하고 자신의 연결 요청
3. ACK: 클라이언트가 서버의 연결 요청 수락

**TCP 4-way Handshake (연결 해제)**
1. FIN: 한쪽이 연결 종료 요청
2. ACK: 상대방이 종료 요청 확인
3. FIN: 상대방도 연결 종료 요청
4. ACK: 마지막 확인

## 주요 프로토콜

### HTTP (HyperText Transfer Protocol)

웹의 기반이 되는 프로토콜로, 클라이언트와 서버 간의 통신을 담당합니다.

**특징:**
- 무상태(Stateless): 각 요청은 독립적
- 요청-응답 모델: 클라이언트가 요청하면 서버가 응답
- 텍스트 기반: 사람이 읽을 수 있는 형태

**실무 팁:**
HTTP는 웹 통신의 기본 프로토콜이다. HTTP/2는 멀티플렉싱을 지원하여 성능이 향상된다.

**HTTP 메서드:**
- GET: 데이터 조회
- POST: 데이터 생성
- PUT: 데이터 수정 (전체 교체)
- PATCH: 데이터 부분 수정
- DELETE: 데이터 삭제

**버전별 특징:**
- HTTP/1.1: 지속 연결, 파이프라이닝, 청크 전송
- HTTP/2: 멀티플렉싱, 서버 푸시, 헤더 압축
- HTTP/3: QUIC 기반, 더 빠른 연결 설정

### HTTPS (HTTP Secure)

HTTP에 보안 기능을 추가한 프로토콜입니다.

**보안 메커니즘:**
- TLS/SSL 암호화: 전송 중인 데이터 암호화
- 인증서 검증: 서버의 신원 확인
- 데이터 무결성: 전송 중 데이터 변조 방지

### WebSocket

실시간 양방향 통신을 위한 프로토콜입니다.

**특징:**
- 전이중 통신: 클라이언트와 서버가 동시에 데이터 전송
- 지속 연결: HTTP 핸드셰이크 후 연결 유지
- 낮은 오버헤드: HTTP 헤더 없이 데이터만 전송

| 특징 | HTTP | WebSocket |
|------|------|-----------|
| 통신 방식 | 요청-응답 | 양방향 |
| 연결 | 일시적 | 지속적 |
| 오버헤드 | 높음 | 낮음 |
| 실시간성 | 제한적 | 우수 |
| 사용 사례 | 웹페이지, API | 채팅, 게임, 알림 |

### TCP (Transmission Control Protocol)

신뢰성이 중요한 데이터 전송에 사용되는 연결 지향 프로토콜입니다.

**주요 기능:**
- 연결 지향: 통신 전에 연결 설정
- 신뢰성: 데이터 손실, 중복, 순서 보장
- 흐름 제어: 수신자의 처리 능력에 맞춰 전송 속도 조절
- 혼잡 제어: 네트워크 상황에 따라 전송 속도 조절

**실무 팁:**
TCP는 신뢰성이 중요한 통신에 사용한다. 웹, 이메일, 파일 전송 등에서 사용한다.

### UDP (User Datagram Protocol)

빠른 전송이 중요한 데이터에 사용되는 비연결 프로토콜입니다.

**특징:**
- 비연결: 연결 설정 없이 바로 데이터 전송
- 빠른 전송: 오류 검사와 재전송 없이 즉시 전송
- 낮은 오버헤드: 최소한의 헤더만 사용
- 순서 보장 없음: 패킷 순서가 바뀔 수 있음

**TCP vs UDP 비교**

| 특징 | TCP | UDP |
|------|-----|-----|
| 연결 방식 | 연결 지향 | 비연결 |
| 신뢰성 | 높음 | 낮음 |
| 속도 | 상대적으로 느림 | 빠름 |
| 오버헤드 | 높음 | 낮음 |
| 순서 보장 | 보장 | 보장 안함 |
| 사용 사례 | 웹, 이메일, 파일 전송 | 스트리밍, 게임, DNS |

## 프로토콜 계층 구조

### OSI 7계층 모델

| 계층 | 이름 | 프로토콜 | 역할 |
|------|------|---------|------|
| 7 | 응용 계층 | HTTP, HTTPS, FTP, SMTP | 애플리케이션과 네트워크 인터페이스 |
| 6 | 표현 계층 | ASCII, JPEG, MPEG | 데이터 형식 변환, 암호화, 압축 |
| 5 | 세션 계층 | NetBIOS, RPC | 통신 세션 설정/관리/해제 |
| 4 | 전송 계층 | TCP, UDP | 종단 간 신뢰성 있는 전송 |
| 3 | 네트워크 계층 | IP, ICMP, OSPF | 패킷 경로 설정과 라우팅 |
| 2 | 데이터 링크 계층 | Ethernet, Wi-Fi | 물리적 링크의 신뢰성 전송 |
| 1 | 물리 계층 | Cable, Hub | 실제 전기 신호 전송 |

### TCP/IP 4계층 모델

- **응용 계층**: HTTP, HTTPS, FTP, SMTP, DNS, SSH
- **전송 계층**: TCP, UDP
- **인터넷 계층**: IP, ICMP, ARP
- **네트워크 액세스 계층**: Ethernet, Wi-Fi, PPP

## 프로토콜 선택 가이드

### 사용 사례별 권장 프로토콜

**신뢰성이 중요한 경우 - TCP**
- 웹 페이지 로딩
- 이메일 전송
- 파일 다운로드
- 데이터베이스 연결

**속도가 중요한 경우 - UDP**
- 실시간 스트리밍 (YouTube, Netflix)
- 온라인 게임
- VoIP 통화
- DNS 조회

**실시간 양방향 통신 - WebSocket**
- 채팅 애플리케이션
- 실시간 알림
- 협업 도구
- 온라인 게임

**보안이 중요한 경우 - HTTPS**
- 온라인 쇼핑
- 인터넷 뱅킹
- 개인정보 처리
- API 통신

## 참고

### 최신 프로토콜 동향

**HTTP/3**
- QUIC 프로토콜: UDP 기반의 새로운 전송 프로토콜
- 빠른 연결 설정: TLS와 연결 설정을 동시에 처리
- 멀티플렉싱: 패킷 손실이 다른 스트림에 영향 주지 않음
- 모바일 최적화: 네트워크 변경 시에도 연결 유지
