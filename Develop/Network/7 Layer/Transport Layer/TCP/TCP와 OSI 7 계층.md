# OSI 7 계층 모델

## OSI 7 계층 모델이란?

### OSI 모델의 정의와 목적

OSI(Open Systems Interconnection) 7 계층 모델은 국제표준화기구(ISO)에서 개발한 네트워크 통신의 표준 모델이다. 복잡한 네트워크 통신 과정을 7개의 계층으로 나누어 각 계층의 역할과 책임을 명확히 정의했다.

### OSI 모델이 필요한 이유

네트워크 통신은 복잡한 과정이다. 한 컴퓨터에서 다른 컴퓨터로 데이터를 전송하려면 다음과 같은 작업들이 필요하다:

- 데이터 형식 변환: 텍스트, 이미지, 동영상 등 다양한 형태의 데이터를 네트워크에서 전송 가능한 형태로 변환
- 암호화/복호화: 보안을 위한 데이터 암호화
- 경로 설정: 데이터가 목적지까지 어떤 경로로 이동할지 결정
- 오류 검출/수정: 전송 중 발생할 수 있는 오류를 찾아서 수정
- 물리적 전송: 실제 전기 신호나 빛 신호로 데이터를 전송

이러한 복잡한 과정을 체계적으로 관리하기 위해 OSI 7 계층 모델이 만들어졌다.

**실무 팁:**
OSI 7 계층 모델은 네트워크 통신을 이해하는 데 유용하다. 각 계층의 역할을 이해하면 네트워크 문제를 진단하는 데 도움이 된다.

### OSI 7 계층 전체 구조

| 계층 | 계층 이름 | 주요 역할 | 간단한 설명 |
|------|-----------|-----------|-------------|
| 7 | 응용 계층<br>(Application Layer) | 사용자와 직접 상호작용 | 웹 브라우저, 이메일 클라이언트 등 사용자가 직접 사용하는 프로그램 |
| 6 | 표현 계층<br>(Presentation Layer) | 데이터 형식 변환 | 이미지, 텍스트 등 다양한 형태의 데이터를 네트워크에서 전송 가능한 형태로 변환 |
| 5 | 세션 계층<br>(Session Layer) | 연결 세션 관리 | 두 컴퓨터 간의 대화(세션)를 시작하고, 유지하고, 종료 |
| 4 | 전송 계층<br>(Transport Layer) | 데이터 전송의 신뢰성 보장 | 데이터가 올바르게 전송되었는지 확인하고, 손실된 데이터를 재전송 |
| 3 | 네트워크 계층<br>(Network Layer) | 데이터 전송 경로 설정 | 인터넷상에서 데이터가 어떤 경로로 이동할지 결정 |
| 2 | 데이터 링크 계층<br>(Data Link Layer) | 물리적 연결과 오류 제어 | 실제 네트워크 장비 간의 연결과 데이터 전송 오류 검출 |
| 1 | 물리 계층<br>(Physical Layer) | 실제 신호 전송 | 전기 신호나 빛 신호로 데이터를 실제로 전송 |

**데이터 캡슐화 과정:**
```
사용자 데이터
  ↓
TCP 세그먼트 (+ TCP 헤더)
  ↓
IP 패킷 (+ IP 헤더)
  ↓
이더넷 프레임 (+ 이더넷 헤더)
  ↓
비트 스트림
```

## 각 계층별 상세 분석

### 7계층: 응용 계층 (Application Layer)

**역할:** 사용자와 직접 상호작용하는 최상위 계층

**주요 기능:**
- 사용자 애플리케이션이 네트워크 서비스에 접근할 수 있도록 함
- 네트워크 서비스의 인터페이스 제공
- 사용자 인증, 데이터 압축, 암호화 등

**대표적인 프로토콜:**
- HTTP/HTTPS: 웹 페이지 요청/응답
- FTP: 파일 전송
- SMTP: 이메일 전송
- POP3/IMAP: 이메일 수신/관리
- DNS: 도메인 이름을 IP 주소로 변환
- SSH: 보안 원격 접속
- Telnet: 원격 접속

**실제 예시:**
- 웹 브라우저에서 구글에 접속할 때
- 이메일 클라이언트로 메일을 보낼 때
- FTP 클라이언트로 파일을 업로드할 때

**실무 팁:**
응용 계층은 사용자가 직접 사용하는 프로토콜이다. HTTP, HTTPS, SSH 등이 여기에 해당한다.

### 6계층: 표현 계층 (Presentation Layer)

**역할:** 데이터 형식 변환, 암호화, 압축

**주요 기능:**
- 데이터의 표현 방식을 통일
- 암호화/복호화 담당
- 압축/해제 기능
- 플랫폼 간 호환성 보장

**대표적인 프로토콜:**
- SSL/TLS: 보안 연결
- JPEG/PNG: 이미지 압축
- ASCII/Unicode: 텍스트 인코딩
- MIME: 멀티미디어 데이터 형식

**실제 예시:**
- 웹사이트의 HTTPS 보안 연결
- 이미지 파일의 압축/해제
- 한글 텍스트의 UTF-8 인코딩

### 5계층: 세션 계층 (Session Layer)

**역할:** 통신 세션의 설정, 관리, 종료

**주요 기능:**
- 통신 세션의 생명주기 관리
- 세션 복구 및 동기화
- 로그인/로그아웃 처리
- 세션 타임아웃 관리

**대표적인 프로토콜:**
- NetBIOS: Windows 네트워크
- RPC: 원격 프로시저 호출
- SQL: 데이터베이스 세션
- NFS: 네트워크 파일 시스템
- SMB: 파일 공유

**실제 예시:**
- 웹사이트에 로그인한 후 세션 유지
- 원격 서버에 접속하여 작업하는 세션
- 파일 서버에 연결하여 파일을 공유하는 세션

### 4계층: 전송 계층 (Transport Layer)

**역할:** 신뢰성 있는 데이터 전송 보장

**주요 기능:**
- 종단간(end-to-end) 통신 담당
- 포트 번호를 통한 프로세스 식별
- 데이터 분할 및 재조립
- 오류 검출 및 복구

**대표적인 프로토콜:**
- TCP: 신뢰성 있는 연결 지향 통신
- UDP: 빠른 비연결 통신
- SCTP: 멀티스트림 전송
- DCCP: 실시간 데이터 전송

**TCP vs UDP 비교:**

| 특징 | TCP | UDP |
|------|-----|-----|
| 연결 방식 | 연결 지향 (전화처럼) | 비연결 지향 (편지처럼) |
| 신뢰성 | 높음 (손실 시 재전송) | 없음 (손실 허용) |
| 순서 보장 | 보장 (순서 번호 사용) | 보장 안함 |
| 속도 | 상대적으로 느림 | 빠름 |
| 오버헤드 | 높음 (헤더 크기 20-60바이트) | 낮음 (헤더 크기 8바이트) |
| 용도 | 웹, 이메일, 파일 전송 | 스트리밍, 게임, DNS |

**실제 예시:**
- 웹 브라우징 (TCP 사용)
- 이메일 전송 (TCP 사용)
- 실시간 스트리밍 (UDP 사용)
- 온라인 게임 (UDP 사용)

**실무 팁:**
TCP는 신뢰성이 중요한 통신에 사용하고, UDP는 속도가 중요한 통신에 사용한다.

### 3계층: 네트워크 계층 (Network Layer)

**역할:** 데이터 전송 경로 설정 및 라우팅

**주요 기능:**
- 논리적 주소(IP 주소) 사용
- 최적 경로 선택
- 패킷 분할 및 재조립
- 인터네트워킹 지원

**대표적인 프로토콜:**
- IP: 인터넷 프로토콜
- ICMP: 인터넷 제어 메시지 프로토콜
- IGMP: 인터넷 그룹 관리 프로토콜
- ARP: 주소 해석 프로토콜
- OSPF: 라우팅 프로토콜
- BGP: 경계 게이트웨이 프로토콜

**실제 예시:**
- 인터넷에서 웹사이트에 접속할 때
- ping 명령어로 네트워크 연결 확인
- 라우터가 패킷을 목적지로 전달

**실무 팁:**
네트워크 계층은 IP 주소를 사용하여 패킷을 목적지로 전달한다. 라우터가 이 계층에서 동작한다.

### 2계층: 데이터 링크 계층 (Data Link Layer)

**역할:** 물리적 연결과 오류 제어

**주요 기능:**
- 물리적 주소(MAC 주소) 사용
- 프레임 단위로 데이터 전송
- 오류 검출 및 수정
- 스위치의 MAC 주소 학습

**대표적인 프로토콜:**
- Ethernet: 유선 네트워크
- WiFi: 무선 네트워크
- PPP: 모뎀 연결
- HDLC: 고급 데이터 링크 제어
- Frame Relay: 프레임 릴레이
- ATM: 비동기 전송 모드

**이더넷 프레임 구조:**
- Preamble: 7바이트 - 동기화
- SFD: 1바이트 - 프레임 시작
- Destination MAC: 6바이트 - 목적지 MAC 주소
- Source MAC: 6바이트 - 출발지 MAC 주소
- Type: 2바이트 - 상위 계층 프로토콜
- Data: 46-1500바이트 - 실제 데이터
- FCS: 4바이트 - 오류 검출 코드

**실제 예시:**
- 스위치가 MAC 주소를 학습하여 패킷을 올바른 포트로 전달
- 이더넷 케이블을 통한 데이터 전송
- WiFi를 통한 무선 데이터 전송

**실무 팁:**
데이터 링크 계층은 MAC 주소를 사용하여 같은 네트워크 내에서 패킷을 전달한다. 스위치가 이 계층에서 동작한다.

### 1계층: 물리 계층 (Physical Layer)

**역할:** 실제 신호 전송

**주요 기능:**
- 비트 단위 데이터 전송
- 전기적, 광학적, 무선 신호 변환
- 케이블, 커넥터, 허브 등 물리적 장비
- 전송 매체의 특성에 따른 신호 조정

**대표적인 프로토콜:**
- RS-232: 직렬 통신
- V.35: 모뎀 연결
- 10BASE-T: 10Mbps 이더넷
- 100BASE-TX: 100Mbps 이더넷
- 1000BASE-T: 1Gbps 이더넷

**신호 유형:**
- 전기 신호: 이더넷 케이블
- 광 신호: 광케이블
- 무선 신호: WiFi, 블루투스
- 적외선 신호: 리모컨

**실제 예시:**
- 이더넷 케이블을 통한 전기 신호 전송
- 광케이블을 통한 빛 신호 전송
- WiFi를 통한 무선 신호 전송

**실무 팁:**
물리 계층은 실제 신호를 전송한다. 케이블, 허브 등이 이 계층에서 동작한다.

## 데이터 캡슐화와 디캡슐화

### 캡슐화(Encapsulation) 과정

데이터가 상위 계층에서 하위 계층으로 전달될 때 각 계층에서 헤더를 추가하는 과정이다.

**캡슐화 과정**:
1. **7계층**: 사용자 데이터 (예: "Hello World")
2. **6계층**: 표현 계층 헤더 추가 (인코딩, 압축, 암호화 정보)
3. **5계층**: 세션 계층 헤더 추가 (세션 ID, 순서 번호)
4. **4계층**: 전송 계층 헤더 추가 (TCP/UDP 헤더 - 포트 번호, 순서 번호)
5. **3계층**: 네트워크 계층 헤더 추가 (IP 헤더 - IP 주소, TTL)
6. **2계층**: 데이터 링크 계층 헤더 추가 (이더넷 헤더 - MAC 주소)
7. **1계층**: 물리 계층 신호 변환 (전기 신호로 변환)

### 디캡슐화(Decapsulation) 과정

수신 측에서 하위 계층에서 상위 계층으로 올라가면서 헤더를 제거하는 과정이다.

**디캡슐화 과정**:
1. **1계층**: 물리적 신호를 디지털 데이터로 변환
2. **2계층**: 이더넷 프레임에서 IP 패킷 추출
3. **3계층**: IP 패킷에서 TCP 세그먼트 추출
4. **4계층**: TCP 세그먼트에서 세션 데이터 추출
5. **5계층**: 세션 데이터에서 표현 데이터 추출
6. **6계층**: 표현 데이터에서 응용 데이터 추출
7. **7계층**: 최종 응용 데이터 처리

### 데이터 단위별 명칭

각 계층에서 사용하는 데이터 단위의 명칭:

| 계층 | 데이터 단위 | 설명 |
|------|-------------|------|
| 7계층 | **메시지(Message)** | 사용자 데이터 |
| 6계층 | **메시지(Message)** | 인코딩/압축된 데이터 |
| 5계층 | **메시지(Message)** | 세션 정보가 추가된 데이터 |
| 4계층 | **세그먼트(Segment)** | TCP/UDP 헤더가 추가된 데이터 |
| 3계층 | **패킷(Packet)** | IP 헤더가 추가된 데이터 |
| 2계층 | **프레임(Frame)** | 이더넷 헤더가 추가된 데이터 |
| 1계층 | **비트(Bit)** | 전기 신호로 변환된 데이터 |

## 4. 계층별 프로토콜과 장비

### OSI 7 계층별 프로토콜과 장비 정리표

| 계층 | 계층 이름 | 주요 프로토콜 | 주요 장비 |
|------|-----------|---------------|-----------|
| **7** | 응용 계층 | HTTP, HTTPS, FTP, SMTP, POP3, IMAP, DNS, SSH, Telnet, SNMP | 웹 서버, 메일 서버, DNS 서버, FTP 서버, 프록시 서버 |
| **6** | 표현 계층 | SSL, TLS, JPEG, PNG, ASCII, Unicode, MIME | 암호화 장비, 압축 장비, 코덱 |
| **5** | 세션 계층 | NetBIOS, RPC, SQL, NFS, SMB | 세션 관리 서버, 인증 서버 |
| **4** | 전송 계층 | TCP, UDP, SCTP, DCCP | 방화벽, 로드 밸런서 (L4), 게이트웨이 |
| **3** | 네트워크 계층 | IP, ICMP, IGMP, ARP, RARP, OSPF, BGP, RIP | 라우터, L3 스위치, 방화벽, 게이트웨이 |
| **2** | 데이터 링크 계층 | Ethernet, WiFi, PPP, HDLC, Frame Relay, ATM | 스위치, 브리지, WiFi 액세스 포인트, 모뎀 |
| **1** | 물리 계층 | RS-232, V.35, 10BASE-T, 100BASE-TX, 1000BASE-T | 허브, 리피터, 케이블, 커넥터, NIC |

### 계층별 주요 장비의 역할

**라우터 (Router)**:
- **동작 계층**: 1, 2, 3계층
- **주요 기능**: IP 주소를 기반으로 패킷을 목적지로 전달
- **라우팅 테이블**: 목적지 네트워크와 다음 홉 정보를 저장

**스위치 (Switch)**:
- **동작 계층**: 1, 2계층
- **주요 기능**: MAC 주소를 기반으로 프레임을 올바른 포트로 전달
- **MAC 주소 테이블**: MAC 주소와 포트 정보를 저장

**방화벽 (Firewall)**:
- **동작 계층**: 1, 2, 3, 4계층 (L7 방화벽은 7계층까지)
- **주요 기능**: 보안 정책에 따라 패킷을 필터링
- **패킷 필터링**: IP 주소, 포트 번호, 프로토콜 등을 기반으로 차단/허용

**로드 밸런서 (Load Balancer)**:
- **L4 로드 밸런서**: 1, 2, 3, 4계층
- **L7 로드 밸런서**: 1, 2, 3, 4, 7계층
- **주요 기능**: 여러 서버에 트래픽을 분산

**허브 (Hub)**:
- **동작 계층**: 1계층
- **주요 기능**: 받은 신호를 모든 포트로 브로드캐스트
- **특징**: 충돌 도메인을 공유하므로 성능이 제한적

## 5. 실제 통신 과정 예시

### 웹 브라우저에서 웹사이트 접속 과정

**1단계: 사용자 입력 (7계층)**
- 사용자가 브라우저에 "https://www.google.com" 입력
- 브라우저가 HTTP 요청 메시지 생성

**2단계: 데이터 변환 (6계층)**
- 텍스트를 UTF-8로 인코딩
- HTTPS를 위한 암호화 준비

**3단계: 세션 관리 (5계층)**
- 웹사이트와의 세션 시작
- 세션 ID 생성 및 관리

**4단계: 신뢰성 있는 전송 (4계층)**
- TCP 연결 설정 (3-Way Handshake)
- 포트 번호 할당 (출발지: 임의, 목적지: 443)

**5단계: 경로 설정 (3계층)**
- DNS 조회로 "www.google.com"을 IP 주소로 변환
- IP 헤더 추가 (출발지 IP, 목적지 IP)

**6단계: 물리적 연결 (2계층)**
- 이더넷 프레임 생성
- MAC 주소 추가 (출발지 MAC, 목적지 MAC)

**7단계: 신호 전송 (1계층)**
- 전기 신호로 변환하여 케이블을 통해 전송

### 이메일 전송 과정

**1단계: 이메일 작성 (7계층)**
- 사용자가 이메일 클라이언트에서 메일 작성
- SMTP 프로토콜 사용

**2단계: 데이터 변환 (6계층)**
- 텍스트 인코딩 (UTF-8)
- 첨부파일 인코딩 (Base64)

**3단계: 세션 관리 (5계층)**
- SMTP 서버와의 세션 시작
- 인증 과정 수행

**4단계: 신뢰성 있는 전송 (4계층)**
- TCP 연결 설정
- 포트 25 사용

**5단계: 경로 설정 (3계층)**
- 수신자 도메인의 메일 서버 IP 주소 조회
- IP 헤더 추가

**6단계: 물리적 연결 (2계층)**
- 이더넷 프레임 생성
- MAC 주소 추가

**7단계: 신호 전송 (1계층)**
- 전기 신호로 변환하여 전송

## 6. TCP/IP 모델과의 비교

### OSI 7 계층 vs TCP/IP 4 계층

| OSI 7 계층 | TCP/IP 4 계층 | 주요 프로토콜 |
|------------|---------------|---------------|
| 7. 응용 계층 | | |
| 6. 표현 계층 | **4. 응용 계층** | HTTP, FTP, SMTP, DNS |
| 5. 세션 계층 | | |
| 4. 전송 계층 | **3. 전송 계층** | TCP, UDP |
| 3. 네트워크 계층 | **2. 인터넷 계층** | IP, ICMP, ARP |
| 2. 데이터 링크 계층 | | |
| 1. 물리 계층 | **1. 네트워크 액세스 계층** | Ethernet, WiFi |

### 두 모델의 차이점

**OSI 7 계층 모델**:
- **장점**: 계층별 역할이 명확히 구분됨
- **단점**: 실제 구현에서는 일부 계층이 통합되어 사용됨
- **용도**: 네트워크 개념 이해와 교육에 주로 사용

**TCP/IP 4 계층 모델**:
- **장점**: 실제 인터넷에서 사용되는 프로토콜 구조
- **단점**: 계층별 구분이 상대적으로 모호함
- **용도**: 실제 네트워크 구현에 사용

### 실제 사용에서의 차이

**OSI 모델**:
- 네트워크 교육과 개념 설명에 주로 사용
- 각 계층의 역할을 명확히 이해할 수 있음
- 네트워크 문제 해결 시 계층별 접근 가능

**TCP/IP 모델**:
- 실제 인터넷 통신에 사용
- 대부분의 네트워크 장비와 소프트웨어가 이 모델을 기반으로 구현
- 실무에서 더 실용적

## 주요 내용 정리

### OSI 7 계층 모델의 주요 개념

1. 계층화된 구조: 복잡한 네트워크 통신을 7개의 계층으로 나누어 관리
2. 각 계층의 독립성: 각 계층은 독립적으로 개발하고 수정 가능
3. 표준화: 국제 표준으로 정의되어 호환성 보장
4. 문제 해결: 네트워크 문제 발생 시 특정 계층에서 해결 가능

### 계층별 주요 역할

- 7계층: 사용자 인터페이스
- 6계층: 데이터 형식 변환
- 5계층: 세션 관리
- 4계층: 신뢰성 있는 전송
- 3계층: 경로 설정
- 2계층: 물리적 연결
- 1계층: 실제 신호 전송

### 실무에서의 활용

- 네트워크 설계: 계층별로 체계적인 네트워크 설계
- 문제 해결: 계층별 접근으로 네트워크 문제 진단
- 보안: 각 계층별 보안 정책 수립

**실무 팁:**
OSI 7 계층 모델은 네트워크 문제를 진단할 때 유용하다. 각 계층별로 문제를 분리하여 해결할 수 있다.
