## 1️⃣ IP 주소의 기본 개념

### 🌐 IP 주소란 무엇인가?

IP(Internet Protocol) 주소는 **인터넷이나 네트워크에서 각 장치를 고유하게 식별하는 숫자 주소**입니다. 마치 집 주소처럼, 네트워크에서 특정 컴퓨터나 기기를 찾기 위한 고유한 식별자 역할을 합니다.

### 🏠 일상생활 비유
- **공인 IP**: 집의 실제 주소 (예: 서울시 강남구 테헤란로 123번지)
- **사설 IP**: 집 안의 방 번호 (예: 201호, 301호)
- **라우터**: 집의 현관문 (외부와 내부를 연결하는 관문)
- **NAT**: 현관문의 인터폰 시스템 (외부에서 내부로 들어오는 것을 제어)

### 🔑 핵심 용어 정리
- **ISP (Internet Service Provider)**: 인터넷 서비스 제공업체 (KT, SKT, LG U+ 등)
- **LAN (Local Area Network)**: 근거리 통신망, 같은 건물이나 지역 내의 네트워크
- **WAN (Wide Area Network)**: 광역 통신망, 인터넷 전체를 의미
- **라우터**: 네트워크 간 데이터를 전달하는 장치
- **서브넷**: 네트워크를 더 작은 단위로 나눈 것

## 2️⃣ IP 주소 구조와 분류

### 🔢 IPv4 주소 구조

IPv4 주소는 **32비트(4바이트)**로 구성되며, **8비트씩 4개 부분(옥텟)**으로 나뉩니다.

```
192.168.001.056
│   │   │   │
│   │   │   └── 호스트 부분 (기기 식별)
│   │   └────── 네트워크 부분
│   └────────── 네트워크 부분
└────────────── 네트워크 부분
```

### 📊 IP 클래스별 구분

| 클래스 | 범위 | 기본 서브넷 마스크 | 용도 | 특징 |
|--------|------|-------------------|------|------|
| **A 클래스** | `1.0.0.0 ~ 126.255.255.255` | `255.0.0.0` | 대규모 네트워크 | 첫 번째 옥텟이 네트워크 ID |
| **B 클래스** | `128.0.0.0 ~ 191.255.255.255` | `255.255.0.0` | 중간 규모 네트워크 | 첫 두 옥텟이 네트워크 ID |
| **C 클래스** | `192.0.0.0 ~ 223.255.255.255` | `255.255.255.0` | 소규모 네트워크 | 첫 세 옥텟이 네트워크 ID |
| **D 클래스** | `224.0.0.0 ~ 239.255.255.255` | - | 멀티캐스트 | 특수 목적용 |
| **E 클래스** | `240.0.0.0 ~ 255.255.255.255` | - | 실험용 | 예약된 주소 |

### 🎯 서브넷 마스크와 CIDR 표기법

#### CIDR 표기법이란?
`192.168.1.56/24`에서 `/24`는 **서브넷 마스크의 비트 수**를 의미합니다.

| CIDR 표기 | 서브넷 마스크 | 네트워크 비트 | 호스트 비트 | 사용 가능한 호스트 |
|-----------|---------------|---------------|-------------|-------------------|
| `/24` | `255.255.255.0` | 24비트 | 8비트 | 254개 (2^8 - 2) |
| `/16` | `255.255.0.0` | 16비트 | 16비트 | 65,534개 (2^16 - 2) |
| `/8` | `255.0.0.0` | 8비트 | 24비트 | 16,777,214개 (2^24 - 2) |

#### 예시: 192.168.1.56/24 분석
- **IP 주소**: 192.168.1.56
- **서브넷 마스크**: 255.255.255.0
- **네트워크 주소**: 192.168.1.0
- **브로드캐스트 주소**: 192.168.1.255
- **사용 가능한 호스트**: 192.168.1.1 ~ 192.168.1.254 (254개)

## 3️⃣ 공인 IP(Public IP) 상세 분석

### 🌍 공인 IP의 정의와 특징

공인 IP는 **인터넷에서 전 세계적으로 유일한 IP 주소**로, 인터넷에 직접 연결되어 외부에서 접근 가능한 주소입니다.

### ✨ 공인 IP의 주요 특징

1. **전 세계 유일성**: 전 세계에서 하나의 장치만이 해당 IP를 가질 수 있음
2. **인터넷 직접 접근**: 외부에서 해당 IP로 직접 접근 가능
3. **ISP 할당**: 인터넷 서비스 제공업체에서 할당받음
4. **유료 서비스**: 공인 IP는 유료로 할당받아야 함
5. **보안 취약성**: 외부에서 직접 접근 가능하므로 보안 관리 필요

### 📋 공인 IP 할당 과정

1. **ISP 요청**: 인터넷 서비스 제공업체에 공인 IP 요청
2. **RIR 할당**: 지역 인터넷 레지스트리(RIR)에서 ISP에 IP 대역 할당
3. **ISP 배정**: ISP가 고객에게 개별 IP 주소 배정
4. **라우팅 등록**: 전 세계 라우팅 테이블에 해당 IP 등록

### 🔍 공인 IP 확인 방법

#### 온라인 서비스 이용
- **whatismyip.com**: 가장 간단한 방법
- **ipify.org**: API 형태로 제공
- **Google 검색**: "내 IP 주소" 검색

#### 명령어 사용
- **Windows**: `ipconfig` 또는 `nslookup myip.opendns.com`
- **Mac/Linux**: `curl ifconfig.me` 또는 `dig +short myip.opendns.com`

### 📊 공인 IP 예시

| 버전 | 주소 예시 | 설명 |
|------|----------|------|
| IPv4 | `203.0.113.45` | 일반적인 공인 IPv4 주소 |
| IPv4 | `8.8.8.8` | Google DNS 서버 |
| IPv6 | `2001:db8::ff00:42:8329` | 차세대 IPv6 주소 |

## 4️⃣ 사설 IP(Private IP) 상세 분석

### 🏠 사설 IP의 정의와 특징

사설 IP는 **내부 네트워크(LAN)에서만 사용되는 IP 주소**로, 인터넷에서 직접 접근할 수 없는 주소입니다.

### ✨ 사설 IP의 주요 특징

1. **내부 네트워크 전용**: LAN 내에서만 통신 가능
2. **인터넷 직접 접근 불가**: NAT나 프록시를 통해서만 인터넷 연결
3. **중복 사용 가능**: 다른 네트워크에서 동일한 IP 사용 가능
4. **무료 사용**: 별도 비용 없이 사용 가능
5. **보안 우수**: 외부에서 직접 접근 불가능

### 📊 사설 IP 주소 대역 (RFC 1918)

| 클래스 | 사설 IP 범위 | 서브넷 마스크 | 사용 가능한 호스트 수 | 용도 |
|--------|------------|--------------|-------------------|------|
| **A 클래스** | `10.0.0.0 ~ 10.255.255.255` | `255.0.0.0` | 16,777,214개 | 대기업, 대학 |
| **B 클래스** | `172.16.0.0 ~ 172.31.255.255` | `255.240.0.0` | 1,048,574개 | 중소기업 |
| **C 클래스** | `192.168.0.0 ~ 192.168.255.255` | `255.255.0.0` | 65,534개 | 가정, 소규모 사무실 |

### 🏢 사설 IP 사용 사례

#### 가정용 네트워크
- **라우터**: 192.168.1.1 (게이트웨이)
- **노트북**: 192.168.1.100
- **스마트폰**: 192.168.1.101
- **스마트TV**: 192.168.1.102

#### 기업용 네트워크
- **개발팀**: 10.1.0.0/24
- **인사팀**: 10.2.0.0/24
- **마케팅팀**: 10.3.0.0/24
- **서버실**: 172.16.0.0/24

### 🔍 사설 IP 확인 방법

#### Windows
```cmd
ipconfig
```

#### Mac/Linux
```bash
ifconfig
# 또는
ip addr show
```

#### 브라우저 (WebRTC 이용)
- 온라인 IP 확인 사이트에서 "로컬 IP" 또는 "내부 IP" 확인

## 5️⃣ 공인 IP vs 사설 IP 비교

### 📊 상세 비교표

| 비교 항목 | 공인 IP (Public IP) | 사설 IP (Private IP) |
|----------|-------------------|-------------------|
| **사용 범위** | 인터넷 전체 | 내부 네트워크(LAN)만 |
| **고유성** | 전 세계에서 유일함 | 여러 네트워크에서 동일 사용 가능 |
| **인터넷 직접 연결** | 가능 | 불가능 (NAT 필요) |
| **할당 주체** | ISP (인터넷 서비스 제공업체) | 네트워크 관리자 또는 라우터 |
| **비용** | 유료 (월 요금) | 무료 |
| **보안** | 외부 접근 가능 (보안 주의) | 외부 접근 불가 (상대적으로 안전) |
| **관리** | ISP가 관리 | 네트워크 관리자가 관리 |
| **변경 가능성** | ISP 변경 시 변경됨 | 네트워크 관리자가 자유롭게 변경 |
| **예시** | `203.0.113.45` | `192.168.1.100` |

### 🎯 언제 어떤 IP를 사용해야 할까?

#### 공인 IP 사용이 필요한 경우
- **웹 서버 운영**: 외부에서 접근 가능해야 함
- **메일 서버**: 외부에서 메일 송수신 필요
- **게임 서버**: 외부 플레이어 접속 필요
- **VPN 서버**: 외부에서 접속 가능해야 함

#### 사설 IP 사용이 적합한 경우
- **내부 네트워크 통신**: 사무실 내 컴퓨터 간 통신
- **개인용 기기**: 노트북, 스마트폰 등
- **보안이 중요한 서버**: 데이터베이스 서버 등
- **비용 절약**: 공인 IP 비용을 절약하고 싶은 경우

## 6️⃣ NAT(Network Address Translation)

### 🔧 NAT의 정의와 목적

NAT는 **사설 IP 주소를 공인 IP 주소로 변환하여 인터넷에 연결하는 기술**입니다.

### 🎯 NAT의 주요 목적

1. **IP 주소 부족 문제 해결**: IPv4 주소 고갈 문제 완화
2. **보안 강화**: 사설 네트워크를 외부로부터 보호
3. **비용 절약**: 하나의 공인 IP로 여러 기기 인터넷 연결
4. **네트워크 관리 단순화**: 내부 네트워크 구조를 외부에 숨김

### 📊 NAT 유형과 특징

| NAT 종류 | 설명 | 매핑 방식 | 사용 사례 |
|---------|------|----------|----------|
| **정적 NAT (Static NAT)** | 사설 IP ↔ 공인 IP를 1:1 매핑 | 고정 매핑 | 웹 서버, 메일 서버 |
| **동적 NAT (Dynamic NAT)** | 여러 개의 사설 IP를 동적으로 공인 IP와 매핑 | 동적 매핑 | 기업 네트워크 |
| **PAT (Port Address Translation)** | 하나의 공인 IP를 여러 장치가 포트로 구분하여 공유 | 포트 기반 매핑 | 가정용 라우터 |

### 🔄 NAT 동작 과정

#### 1단계: 내부에서 외부로 (Outbound)
```
[PC: 192.168.1.100:3000] 
    ↓
[라우터: NAT 변환] 
    ↓
[공인 IP: 203.0.113.45:1024] 
    ↓
[인터넷]
```

#### 2단계: 외부에서 내부로 (Inbound)
```
[인터넷] 
    ↓
[공인 IP: 203.0.113.45:1024] 
    ↓
[라우터: NAT 역변환] 
    ↓
[PC: 192.168.1.100:3000]
```

### 🏠 가정용 NAT 시나리오

#### 네트워크 구성
- **공인 IP**: 203.0.113.45 (ISP에서 할당)
- **라우터**: 192.168.1.1 (게이트웨이)
- **기기들**:
  - 노트북: 192.168.1.100
  - 스마트폰: 192.168.1.101
  - 스마트TV: 192.168.1.102

#### NAT 변환 예시
| 내부 기기 | 내부 주소 | 변환된 공인 주소 | 목적지 |
|----------|----------|----------------|--------|
| 노트북 | 192.168.1.100:3000 | 203.0.113.45:1024 | Google.com |
| 스마트폰 | 192.168.1.101:8080 | 203.0.113.45:1025 | YouTube.com |
| 스마트TV | 192.168.1.102:5000 | 203.0.113.45:1026 | Netflix.com |

## 7️⃣ 실제 활용 사례와 시나리오

### 🏢 기업 환경

#### 대기업 네트워크 구성
```
인터넷
    ↓
방화벽 (공인 IP: 203.0.113.10)
    ↓
DMZ (공인 IP: 203.0.113.11-20)
    ↓
내부 네트워크 (사설 IP: 10.0.0.0/8)
    ├── 개발팀: 10.1.0.0/24
    ├── 인사팀: 10.2.0.0/24
    ├── 마케팅팀: 10.3.0.0/24
    └── 서버실: 10.10.0.0/24
```

#### 보안 계층
1. **외부 네트워크**: 인터넷 (공인 IP)
2. **DMZ**: 웹 서버, 메일 서버 (공인 IP)
3. **내부 네트워크**: 업무용 PC, 데이터베이스 (사설 IP)

### ☁️ 클라우드 환경 (AWS VPC)

#### VPC 네트워크 구성
```
VPC: 10.0.0.0/16
├── 퍼블릭 서브넷: 10.0.1.0/24
│   ├── 웹 서버 (Elastic IP)
│   └── 로드 밸런서 (Elastic IP)
└── 프라이빗 서브넷: 10.0.10.0/24
    ├── 애플리케이션 서버
    └── 데이터베이스 서버
    └── NAT Gateway (10.0.1.100)
```

#### NAT Gateway 동작
- **프라이빗 서브넷**의 인스턴스들이 **NAT Gateway**를 통해 인터넷에 접속
- **아웃바운드**만 가능 (외부에서 프라이빗 서브넷으로 직접 접근 불가)
- **보안 강화**: 데이터베이스는 인터넷에 직접 노출되지 않음

### 🏠 가정 환경

#### 일반적인 가정용 네트워크
```
인터넷 (ISP)
    ↓
모뎀 (공인 IP: 203.0.113.45)
    ↓
라우터 (192.168.1.1)
    ├── 노트북 (192.168.1.100)
    ├── 스마트폰 (192.168.1.101)
    ├── 스마트TV (192.168.1.102)
    └── 게임기 (192.168.1.103)
```

#### 포트 포워딩 설정
- **게임 서버 운영**: 192.168.1.103:7777 → 203.0.113.45:7777
- **웹 서버 운영**: 192.168.1.100:80 → 203.0.113.45:8080
- **원격 데스크톱**: 192.168.1.100:3389 → 203.0.113.45:3389

### 📱 모바일 환경

#### 4G/5G 네트워크
```
모바일 기기
    ↓
기지국 (사설 IP 할당)
    ↓
통신사 네트워크 (NAT 적용)
    ↓
인터넷 (공인 IP)
```

#### 모바일 IP 특징
- **동적 할당**: 기지국에서 임시 사설 IP 할당
- **자동 NAT**: 통신사에서 자동으로 NAT 처리
- **위치 이동**: 기지국 변경 시 IP 주소 변경 가능

## 8️⃣ 보안 고려사항

### 🛡️ 공인 IP 보안 관리

#### 주요 위험 요소
1. **직접 공격**: 외부에서 직접 접근 가능
2. **포트 스캔**: 공격자가 열린 포트 탐지
3. **DDoS 공격**: 대량의 트래픽으로 서비스 마비
4. **침입 시도**: 다양한 공격 벡터 존재

#### 보안 대책
1. **방화벽 설정**: 불필요한 포트 차단
2. **포트 포워딩 최소화**: 필요한 포트만 개방
3. **정기적인 보안 업데이트**: 시스템 및 소프트웨어 업데이트
4. **침입 탐지 시스템**: IDS/IPS 구축
5. **VPN 사용**: 원격 접속 시 VPN 활용

### 🔒 사설 IP 보안 관리

#### 상대적 안전성
- **외부 직접 접근 불가**: NAT를 통해서만 접근 가능
- **내부 네트워크 보호**: 외부 공격으로부터 차단
- **네트워크 분리**: 다른 네트워크와 격리

#### 주의사항
1. **내부 공격**: 내부 사용자의 악의적 행위
2. **악성코드**: 내부 네트워크 내 악성코드 확산
3. **권한 관리**: 내부 사용자 권한 적절히 관리
4. **네트워크 모니터링**: 내부 트래픽 모니터링

### 🔐 NAT 보안 기능

#### NAT의 보안 효과
1. **IP 주소 숨김**: 내부 네트워크 구조를 외부에 노출하지 않음
2. **세션 관리**: 연결 상태 추적 및 관리
3. **포트 동적 할당**: 임시 포트 사용으로 예측 어려움
4. **연결 제어**: 아웃바운드 연결만 허용

#### NAT 우회 공격
1. **STUN 서버**: NAT 뒤의 클라이언트 IP 탐지
2. **TURN 서버**: 중계 서버를 통한 연결
3. **UPnP**: 자동 포트 포워딩 기능 악용
4. **피어투피어**: P2P 통신을 통한 직접 연결

## 🎯 핵심 정리

### 📝 공인 IP와 사설 IP의 핵심 차이점

1. **사용 범위**
   - 공인 IP: 인터넷 전체에서 사용
   - 사설 IP: 내부 네트워크에서만 사용

2. **고유성**
   - 공인 IP: 전 세계에서 유일
   - 사설 IP: 다른 네트워크에서 중복 사용 가능

3. **접근성**
   - 공인 IP: 외부에서 직접 접근 가능
   - 사설 IP: NAT를 통해서만 인터넷 연결

4. **비용**
   - 공인 IP: ISP에서 유료 할당
   - 사설 IP: 무료 사용 가능

5. **보안**
   - 공인 IP: 보안 관리 필요
   - 사설 IP: 상대적으로 안전

### 🔧 NAT의 핵심 역할

1. **IP 주소 절약**: 하나의 공인 IP로 여러 기기 연결
2. **보안 강화**: 내부 네트워크를 외부로부터 보호
3. **비용 절약**: 공인 IP 비용 절약
4. **관리 단순화**: 내부 네트워크 구조 숨김

### 💡 실무 적용 가이드

1. **웹 서버**: 공인 IP 필요 (외부 접근)
2. **데이터베이스**: 사설 IP 사용 (보안)
3. **개발 환경**: 사설 IP + 포트 포워딩
4. **클라우드**: VPC 내 사설 IP + NAT Gateway
5. **가정용**: 사설 IP + 라우터 NAT

이 가이드를 통해 공인 IP와 사설 IP의 개념을 명확히 이해하고, 실제 환경에서 적절히 활용할 수 있기를 바랍니다.
