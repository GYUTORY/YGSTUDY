# OSI 7 계층

## 배경

OSI(Open Systems Interconnection) 7 계층은 국제표준화기구(ISO)에서 정의한 네트워크 통신의 표준 모델입니다. 네트워크 통신을 7개의 계층으로 나누어 각 계층의 역할과 책임을 명확히 정의하고, 서로 다른 시스템 간의 통신을 표준화합니다.

### OSI 7 계층의 필요성
- **표준화**: 서로 다른 제조사의 장비 간 통신 보장
- **모듈화**: 각 계층의 독립적인 개발과 수정 가능
- **문제 해결**: 네트워크 문제의 계층별 분리와 해결
- **교육**: 네트워크 개념의 체계적 이해

### 기본 개념
- **캡슐화**: 데이터 전송 시 각 계층에서 헤더 추가
- **디캡슐화**: 데이터 수신 시 각 계층에서 헤더 제거
- **프로토콜**: 각 계층에서 사용하는 통신 규약
- **인터페이스**: 계층 간의 표준화된 연결 지점

## 핵심

### 1. 물리 계층 (Physical Layer) - 1계층

#### 역할과 기능
물리 계층은 실제 데이터를 전기적 신호로 변환하여 물리적 매체를 통해 전송하는 역할을 담당합니다.

**주요 기능**:
- 비트 스트림을 전기적 신호로 변환
- 물리적 매체를 통한 비트 전송
- 물리적 연결의 설정, 유지, 해제
- 전송 속도, 전송 모드(단방향, 양방향) 제어

#### 물리적 매체
```text
- 이더넷 케이블(RJ-45)
- 광섬유 케이블
- 무선 LAN의 전파
- USB 케이블
- 동축 케이블
```

#### 프로토콜과 장비
```text
- 프로토콜: Ethernet, Wi-Fi, Bluetooth
- 장비: 허브(Hub), 리피터(Repeater)
- 단위: 비트(Bit)
```

### 2. 데이터 링크 계층 (Data Link Layer) - 2계층

#### 역할과 기능
데이터 링크 계층은 물리 계층에서 전송된 비트들을 프레임으로 구성하고, 인접한 노드 간의 신뢰성 있는 데이터 전송을 보장합니다.

**주요 기능**:
- 프레임 동기화
- 흐름 제어(Flow Control)
- 오류 제어(Error Control)
- MAC 주소를 통한 통신
- 프레임의 순서 제어

#### 프로토콜
```text
- Ethernet
- PPP(Point-to-Point Protocol)
- HDLC(High-Level Data Link Control)
- ATM(Asynchronous Transfer Mode)
- Wi-Fi (IEEE 802.11)
```

#### 장비와 단위
```text
- 장비: 스위치(Switch), 브리지(Bridge)
- 단위: 프레임(Frame)
- 주소: MAC 주소
```

### 3. 네트워크 계층 (Network Layer) - 3계층

#### 역할과 기능
네트워크 계층은 서로 다른 네트워크 간의 통신을 담당하며, 최적의 경로를 찾아 패킷을 전송합니다.

**주요 기능**:
- 라우팅(Routing)
- 패킷 포워딩(Packet Forwarding)
- 논리적 주소 지정(IP 주소)
- 서브네팅(Subnetting)
- QoS(Quality of Service) 관리

#### 프로토콜
```text
- IP(Internet Protocol)
- ICMP(Internet Control Message Protocol)
- IGMP(Internet Group Management Protocol)
- ARP(Address Resolution Protocol)
- RARP(Reverse Address Resolution Protocol)
- OSPF, BGP (라우팅 프로토콜)
```

#### 장비와 단위
```text
- 장비: 라우터(Router), L3 스위치
- 단위: 패킷(Packet)
- 주소: IP 주소
```

### 4. 전송 계층 (Transport Layer) - 4계층

#### 역할과 기능
전송 계층은 종단 간(end-to-end) 통신을 담당하며, 데이터의 신뢰성 있는 전송을 보장합니다.

**주요 기능**:
- 연결 지향 통신(TCP)과 비연결 통신(UDP) 제공
- 포트 번호를 통한 프로세스 식별
- 흐름 제어와 혼잡 제어
- 오류 검출 및 복구
- 데이터 분할과 재조립

#### 프로토콜
```text
- TCP(Transmission Control Protocol)
- UDP(User Datagram Protocol)
- SCTP(Stream Control Transmission Protocol)
- DCCP(Datagram Congestion Control Protocol)
```

#### 특징 비교
| 프로토콜 | 연결 방식 | 신뢰성 | 속도 | 사용 예시 |
|----------|-----------|--------|------|-----------|
| **TCP** | 연결 지향 | 높음 | 상대적 느림 | 웹, 이메일, 파일 전송 |
| **UDP** | 비연결 | 낮음 | 빠름 | 스트리밍, 게임, DNS |

### 5. 세션 계층 (Session Layer) - 5계층

#### 역할과 기능
세션 계층은 응용 프로그램 간의 대화를 관리하고, 세션의 설정, 유지, 종료를 담당합니다.

**주요 기능**:
- 세션 설정, 관리, 종료
- 동기화(Synchronization)
- 대화 제어(Dialog Control)
- 토큰 관리
- 체크포인트 설정

#### 프로토콜
```text
- NetBIOS
- RPC(Remote Procedure Call)
- SQL
- NFS(Network File System)
- SSH(Secure Shell)
```

### 6. 표현 계층 (Presentation Layer) - 6계층

#### 역할과 기능
표현 계층은 데이터의 형식을 변환하고, 암호화/복호화를 담당하여 응용 계층에서 사용할 수 있는 형태로 데이터를 제공합니다.

**주요 기능**:
- 데이터 형식 변환
- 데이터 압축/해제
- 데이터 암호화/복호화
- 문자 코드 변환
- 미디어 포맷 변환

#### 프로토콜과 포맷
```text
- JPEG, PNG, GIF (이미지 포맷)
- MPEG, AVI (비디오 포맷)
- ASCII, Unicode (문자 인코딩)
- SSL/TLS (암호화)
- MIME (이메일 첨부파일)
```

### 7. 응용 계층 (Application Layer) - 7계층

#### 역할과 기능
응용 계층은 사용자와 직접 상호작용하는 계층으로, 다양한 네트워크 서비스를 제공합니다.

**주요 기능**:
- 사용자 인터페이스 제공
- 네트워크 서비스 구현
- 데이터베이스 접근
- 파일 전송
- 이메일 송수신

#### 프로토콜
```text
- HTTP/HTTPS (웹 서비스)
- FTP (파일 전송)
- SMTP, POP3, IMAP (이메일)
- DNS (도메인 이름 시스템)
- DHCP (동적 IP 할당)
- SNMP (네트워크 관리)
- Telnet, SSH (원격 접속)
```

#### 응용 프로그램
```text
- 웹 브라우저
- 이메일 클라이언트
- 파일 공유 프로그램
- 메신저
- 게임 클라이언트
```

## 예시

### 1. 실제 통신 과정

#### 웹 브라우저에서 웹사이트 접속 과정
```text
1. 응용 계층: HTTP 요청 생성
   GET /index.html HTTP/1.1
   Host: www.example.com

2. 표현 계층: 데이터 인코딩
   ASCII/UTF-8 인코딩

3. 세션 계층: 세션 설정
   TCP 연결 설정

4. 전송 계층: TCP 세그먼트 생성
   [TCP 헤더][HTTP 데이터]

5. 네트워크 계층: IP 패킷 생성
   [IP 헤더][TCP 세그먼트]

6. 데이터 링크 계층: 이더넷 프레임 생성
   [이더넷 헤더][IP 패킷][FCS]

7. 물리 계층: 전기적 신호로 변환
   10101010... (비트 스트림)
```

#### 데이터 캡슐화 과정
```text
응용 계층: HTTP 데이터
    ↓ (HTTP 헤더 추가)
표현 계층: HTTP 메시지
    ↓ (세션 정보 추가)
세션 계층: 세션 데이터
    ↓ (TCP 헤더 추가)
전송 계층: TCP 세그먼트
    ↓ (IP 헤더 추가)
네트워크 계층: IP 패킷
    ↓ (이더넷 헤더 추가)
데이터 링크 계층: 이더넷 프레임
    ↓ (비트로 변환)
물리 계층: 전기적 신호
```

### 2. 계층별 문제 해결

#### 네트워크 문제 진단
```text
문제: 웹사이트 접속 불가

1. 물리 계층 확인
   - 케이블 연결 상태
   - 네트워크 카드 동작

2. 데이터 링크 계층 확인
   - MAC 주소 확인
   - 스위치 포트 상태

3. 네트워크 계층 확인
   - IP 주소 설정
   - 라우팅 테이블

4. 전송 계층 확인
   - 포트 상태
   - 방화벽 설정

5. 응용 계층 확인
   - 웹 서버 동작
   - DNS 설정
```

## 운영 팁

### 네트워크 모니터링

#### 계층별 모니터링 도구
```bash
# 물리 계층
ping -c 4 8.8.8.8

# 데이터 링크 계층
arp -a

# 네트워크 계층
traceroute google.com

# 전송 계층
netstat -tuln

# 응용 계층
nslookup google.com
```

### 성능 최적화

#### 계층별 최적화 전략
```text
1. 물리 계층: 고속 케이블 사용
2. 데이터 링크 계층: 스위치 최적화
3. 네트워크 계층: 라우팅 최적화
4. 전송 계층: TCP 튜닝
5. 응용 계층: 캐싱 활용
```

## 참고

### OSI vs TCP/IP 모델 비교

| OSI 7 계층 | TCP/IP 4 계층 | 프로토콜 |
|------------|---------------|----------|
| **응용 계층** | **응용 계층** | HTTP, FTP, SMTP |
| **표현 계층** | | SSL/TLS, MIME |
| **세션 계층** | | NetBIOS, RPC |
| **전송 계층** | **전송 계층** | TCP, UDP |
| **네트워크 계층** | **인터넷 계층** | IP, ICMP |
| **데이터 링크 계층** | **네트워크 액세스 계층** | Ethernet, Wi-Fi |
| **물리 계층** | | 케이블, 무선 |

### 계층별 보안 고려사항

| 계층 | 보안 위협 | 대응 방안 |
|------|-----------|-----------|
| **응용 계층** | 악성 코드, SQL 인젝션 | 방화벽, WAF |
| **전송 계층** | 세션 하이재킹 | SSL/TLS |
| **네트워크 계층** | IP 스푸핑 | IPsec |
| **데이터 링크 계층** | MAC 스푸핑 | 포트 보안 |
| **물리 계층** | 물리적 접근 | 접근 제어 |

### 결론
OSI 7 계층은 네트워크 통신의 표준 모델로, 각 계층의 역할과 책임을 명확히 정의합니다.
캡슐화와 디캡슐화를 통해 데이터가 각 계층을 통과하며 적절한 헤더가 추가/제거됩니다.
각 계층은 독립적으로 개발되고 수정될 수 있어 모듈화된 네트워크 시스템 구축이 가능합니다.
네트워크 문제 해결 시 계층별 접근을 통해 효율적인 진단과 해결이 가능합니다.
