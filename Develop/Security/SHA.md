# SHA (Secure Hash Algorithm)

## 개요

SHA는 1993년 미국 연방정보처리표준 규격(FIPS PUB 180)으로 발표된 해시 알고리즘이다.

## SHA-256 과정


![sha256-02.png](..%2F..%2Fetc%2Fimage%2FSECURITY%2Fsha256-02.png) 


### 1. 문자열을 바이너리로 변환

문자열을 ASCII 코드로 변환한 뒤 바이너리로 표현한다. 예를 들어 'a'는 ASCII 코드 97이고, 8비트 바이너리로는 01100001이다.

```
01101000 01100101 01101100 01101100 01101111 00100000 01110111 01101111
01110010 01101100 01100100
```

### 2. 바이너리 데이터를 512비트 블록으로 나누기 및 패딩

이진 데이터를 512비트 블록으로 나눈다. 블록이 512비트보다 작으면 패딩을 추가한다. 패딩은 항상 1로 시작하고 나머지 비트는 0으로 채운다.
```
01101000 01100101 01101100 01101100 01101111 00100000 01110111 01101111
01110010 01101100 01100100 10000000 00000000 00000000 00000000 00000000
00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
00000000 00000000 00000000 00000000 00000000 00000000 00000000 00000000
```
나머지 부분은 원래 데이터의 길이를 나타낸다. 패딩 후 비트 수가 448비트가 되도록 0을 채운다. 마지막 64비트는 제외하고 모두 0으로 채워 총 448비트가 된다.


### 3. 512비트 블록을 32비트 워드로 나누기

512비트 블록을 32비트 워드로 나눈다. 32비트 워드는 총 16개가 된다. 각 워드는 32비트 이진수로 표현된다.

### 4. 압축 함수를 64회 반복 수행

압축 함수를 64회 반복해 블록을 처리한다.

### 5. 블록 간 해시 연산

여러 블록이 있으면 각 블록의 해시를 연산한다.

### 6. 최종 해시값 생성

최종적으로 256비트의 해시값이 생성된다. 이것이 SHA-256 결과값이다.

## 사용 예시

```javascript
const crypto = require('crypto');

const data = 'Hello, World!';
const hash = crypto.createHash('sha256').update(data).digest('hex');
console.log(hash); // SHA-256 해시값
```

**실무 팁:**
SHA-256은 데이터 무결성 검증에 사용한다. 비밀번호 해싱에는 bcrypt나 Argon2를 사용하는 게 좋다.


## 참고

- SHA-256은 일방향 함수로 복호화가 불가능하다
- 같은 입력은 항상 같은 출력을 생성한다
- 작은 입력 변경도 완전히 다른 해시값을 생성한다 (눈사태 효과)
