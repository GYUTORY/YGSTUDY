# Node.js 캐싱 전략

## 개요

캐싱은 비싼 연산(DB 쿼리, 외부 API 호출, 복잡한 계산)의 결과를 저장해두고 재사용하는 기법이다. 적절한 캐싱은 응답 시간을 수십 배 줄이고 DB 부하를 대폭 감소시킨다.

```
캐싱이 필요한 신호:
  - 동일한 DB 쿼리가 반복적으로 실행된다
  - 자주 바뀌지 않는 데이터에 매번 조회가 발생한다
  - 외부 API 호출이 느리고 요금이 발생한다
  - CPU 집약적인 계산이 반복된다
```

## 캐시 종류

| 종류 | 위치 | 속도 | 용량 | 공유 |
|------|------|------|------|------|
| **In-Process** | 앱 메모리 | 최고 | 제한 | 인스턴스 내 |
| **Redis** | 외부 서버 | 빠름 | 큼 | 전체 공유 |
| **CDN/HTTP** | 네트워크 엣지 | 매우 빠름 | 큼 | 글로벌 |
| **DB 쿼리 캐시** | DB 내부 | 빠름 | 보통 | DB 공유 |

## In-Process 캐싱

### node-cache

```typescript
import NodeCache from 'node-cache';

const cache = new NodeCache({
  stdTTL: 300,        // 기본 TTL 5분
  checkperiod: 60,    // 만료 체크 주기 (초)
  useClones: false,   // 성능을 위해 clone 비활성화 (객체 불변성 주의)
});

// 기본 사용
cache.set('user:123', userObject);
const user = cache.get<User>('user:123');

// TTL 개별 지정
cache.set('config', configData, 3600); // 1시간
```

### 캐시 래퍼 패턴

```typescript
// utils/cache.ts
export class CacheManager {
  constructor(private readonly cache: NodeCache) {}

  async getOrSet<T>(
    key: string,
    factory: () => Promise<T>,
    ttl?: number
  ): Promise<T> {
    const cached = this.cache.get<T>(key);
    if (cached !== undefined) return cached;

    const data = await factory();
    this.cache.set(key, data, ttl ?? 300);
    return data;
  }

  invalidate(pattern: string): void {
    const keys = this.cache.keys().filter(k => k.startsWith(pattern));
    this.cache.del(keys);
  }
}

// 사용
const userCache = new CacheManager(cache);

async function getUser(id: string): Promise<User> {
  return userCache.getOrSet(
    `user:${id}`,
    () => userRepository.findById(id),
    600 // 10분
  );
}
```

## Redis 캐싱

### 기본 설정

```typescript

const redis = new Redis({

  retryStrategy: (times) => Math.min(times * 50, 2000),
  maxRetriesPerRequest: 3,
});

```

### Redis 캐시 서비스

```typescript
// cache/RedisCache.ts
import redis from './redis';

export class RedisCache {
  async get<T>(key: string): Promise<T | null> {
    const data = await redis.get(key);
    return data ? JSON.parse(data) : null;
  }

  async set(key: string, value: unknown, ttlSeconds: number): Promise<void> {
    await redis.setex(key, ttlSeconds, JSON.stringify(value));
  }

  async del(key: string): Promise<void> {
    await redis.del(key);
  }

  async delPattern(pattern: string): Promise<void> {
    // SCAN으로 패턴 매칭 키 삭제 (KEYS는 프로덕션 금지)
    const keys: string[] = [];
    let cursor = '0';
    do {
      const [nextCursor, found] = await redis.scan(cursor, 'MATCH', pattern, 'COUNT', 100);
      cursor = nextCursor;
      keys.push(...found);
    } while (cursor !== '0');

    if (keys.length > 0) {
      await redis.del(...keys);
    }
  }

  async getOrSet<T>(
    key: string,
    factory: () => Promise<T>,
    ttlSeconds: number
  ): Promise<T> {
    const cached = await this.get<T>(key);
    if (cached !== null) return cached;

    const data = await factory();
    await this.set(key, data, ttlSeconds);
    return data;
  }
}
```

## 캐시 무효화 전략

### 1. TTL 기반 (Time-To-Live)

```typescript
// 단순하지만 stale 데이터가 잠시 허용됨
await cache.set('product:list', products, 300); // 5분 후 자동 만료
```

### 2. Write-Through (쓸 때 같이 업데이트)

```typescript
async function updateUser(id: string, data: UpdateUserDto): Promise<User> {
  const user = await userRepository.update(id, data);

  // DB 업데이트와 동시에 캐시 갱신
  await cache.set(`user:${id}`, user, 600);

  return user;
}
```

### 3. Cache-Aside (Lazy Loading)

```typescript
async function getProduct(id: string): Promise<Product> {
  // 1. 캐시 확인
  const cached = await cache.get<Product>(`product:${id}`);
  if (cached) return cached;

  // 2. 캐시 미스 → DB 조회
  const product = await productRepository.findById(id);
  if (!product) throw new NotFoundError('상품');

  // 3. 캐시 저장
  await cache.set(`product:${id}`, product, 300);

  return product;
}
```

### 4. 이벤트 기반 무효화

```typescript
// 상품 수정 이벤트 발생 시 관련 캐시 전체 삭제
eventEmitter.on('product.updated', async (productId: string) => {
  await Promise.all([
    cache.del(`product:${productId}`),
    cache.delPattern('product:list:*'), // 목록 캐시 전체 무효화
  ]);
});
```

## 캐시 키 설계

```typescript
// 일관된 키 네이밍 규칙
class CacheKeys {
  static user(id: string) { return `user:${id}`; }
  static userList(page: number, limit: number) { return `user:list:${page}:${limit}`; }
  static product(id: string) { return `product:${id}`; }
  static productsByCategory(categoryId: string) { return `product:category:${categoryId}`; }

// 사용
await cache.get(CacheKeys.user('123'));
await cache.delPattern('product:*'); // 상품 관련 캐시 전체 삭제
```

## 주의사항

### Cache Stampede (thundering herd)

```typescript
// 캐시 만료 시 동시에 다수 요청이 DB를 공격하는 문제
// 해결: mutex (잠금) 사용

import { Mutex } from 'async-mutex';
const mutex = new Mutex();

async function getWithLock<T>(key: string, factory: () => Promise<T>): Promise<T> {
  const cached = await cache.get<T>(key);
  if (cached !== null) return cached;

  return mutex.runExclusive(async () => {
    // 잠금 후 다시 확인 (이미 다른 요청이 채웠을 수 있음)
    const afterLock = await cache.get<T>(key);
    if (afterLock !== null) return afterLock;

    const data = await factory();
    await cache.set(key, data, 300);
    return data;
  });
}
```

### 캐시해서는 안 되는 것

```
❌ 캐시 금지:
  - 사용자 비밀번호, 토큰, 개인정보
  - 실시간 재고, 잔액 등 정확도가 중요한 데이터
  - 사용자별 권한 정보 (보안 취약점 가능)

✅ 캐시 적합:
  - 카테고리 목록, 공통 설정
  - 자주 조회되고 잘 바뀌지 않는 데이터
  - 외부 API 응답 (날씨, 환율 등)
  - 집계 결과 (통계, 대시보드)
```

## 성능 모니터링

```typescript
// 캐시 히트율 추적
let hits = 0;
let misses = 0;

async function trackedGet<T>(key: string): Promise<T | null> {
  const result = await cache.get<T>(key);
  if (result !== null) hits++;
  else misses++;
  return result;
}

// 히트율 로깅 (60초마다)
setInterval(() => {
  const total = hits + misses;
  const hitRate = total > 0 ? ((hits / total) * 100).toFixed(1) : '0';
  logger.info('Cache hit rate', { hitRate: `${hitRate}%`, hits, misses });
  hits = 0;
  misses = 0;
}, 60_000);
```
