# 마이크로서비스 통신 패턴

## 개요

마이크로서비스 간 통신은 크게 **동기(Synchronous)** 와 **비동기(Asynchronous)** 두 가지로 나뉜다. 어떤 방식을 선택하느냐에 따라 서비스의 결합도, 가용성, 복잡도가 달라진다.

```
동기 통신: 요청 → 응답을 즉시 기다림
  - REST, gRPC
  - 응답이 필요할 때, 즉각적인 피드백이 필요할 때

비동기 통신: 요청 후 응답을 기다리지 않음
  - 메시지 큐 (Kafka, RabbitMQ), 이벤트 브로커
  - 느슨한 결합, 높은 가용성이 필요할 때
```

## 동기 통신 패턴

### REST

```typescript
// HTTP 클라이언트 — axios + 재시도 로직
import axios, { AxiosInstance } from 'axios';
import axiosRetry from 'axios-retry';

function createHttpClient(baseURL: string): AxiosInstance {
  const client = axios.create({
    baseURL,
    timeout: 5000,
    headers: { 'Content-Type': 'application/json' },
  });

  axiosRetry(client, {
    retries: 3,
    retryDelay: axiosRetry.exponentialDelay,
    retryCondition: (error) =>
      axiosRetry.isNetworkOrIdempotentRequestError(error) ||
      error.response?.status === 503,
  });

  // 서비스 간 요청 ID 전파
  client.interceptors.request.use((config) => {
    const requestId = (global as any).requestId;
    if (requestId) config.headers['x-request-id'] = requestId;
    return config;
  });

  return client;
}

// 사용
const userClient = createHttpClient('http://user-service:3001');

async function getUserById(id: string) {
  const { data } = await userClient.get(`/users/${id}`);
  return data;
}
```

### gRPC

```typescript
// proto/user.proto 정의 후 사용
import * as grpc from '@grpc/grpc-js';
import * as protoLoader from '@grpc/proto-loader';

const packageDef = protoLoader.loadSync('proto/user.proto', {
  keepCase: true,
  longs: String,
  enums: String,
});
const proto = grpc.loadPackageDefinition(packageDef) as any;

const client = new proto.user.UserService(
  'user-service:50051',
  grpc.credentials.createInsecure()
);

// Promise 래퍼
function getUserById(id: string): Promise<User> {
  return new Promise((resolve, reject) => {
    client.GetUser({ id }, (err: Error | null, response: User) => {
      if (err) reject(err);
      else resolve(response);
    });
  });
}
```

### Circuit Breaker

동기 통신의 핵심 패턴 — 하위 서비스 장애가 전체로 전파되는 것을 차단.

```typescript
import { CircuitBreaker } from 'opossum';

const options = {
  timeout: 3000,            // 3초 내 응답 없으면 실패
  errorThresholdPercentage: 50, // 50% 이상 실패 시 열림(Open)
  resetTimeout: 10000,      // 10초 후 Half-Open 시도
};

const breaker = new CircuitBreaker(getUserById, options);

breaker.on('open', () => logger.warn('Circuit breaker opened: user-service'));
breaker.on('halfOpen', () => logger.info('Circuit breaker half-open'));
breaker.on('close', () => logger.info('Circuit breaker closed'));

// 사용
try {
  const user = await breaker.fire(userId);
} catch (err) {
  // fallback 처리
  return { id: userId, name: 'Unknown', cached: true };
}
```

## 비동기 통신 패턴

### Kafka 이벤트 발행/구독

```typescript
// events/publisher.ts
import { Kafka } from 'kafkajs';

const kafka = new Kafka({
  clientId: 'order-service',
  brokers: (process.env.KAFKA_BROKERS ?? 'localhost:9092').split(','),
});

const producer = kafka.producer();

export async function publishEvent<T>(topic: string, event: T): Promise<void> {
  await producer.send({
    topic,
    messages: [{
      key: (event as any).id ?? null,
      value: JSON.stringify(event),
      headers: { 'event-type': topic },
    }],
  });
}

// 사용
await publishEvent('order.created', {
  id: orderId,
  userId,
  items,
  totalAmount,
  createdAt: new Date().toISOString(),
});
```

```typescript
// events/consumer.ts
const consumer = kafka.consumer({ groupId: 'payment-service' });

await consumer.subscribe({ topics: ['order.created'], fromBeginning: false });

await consumer.run({
  eachMessage: async ({ topic, partition, message }) => {
    const event = JSON.parse(message.value!.toString());
    logger.info('Event received', { topic, eventId: event.id });

    try {
      await handleOrderCreated(event);
    } catch (err) {
      logger.error('Event handling failed', { topic, eventId: event.id, error: err });
      // Dead Letter Queue로 보내거나 재시도 로직 추가
    }
  },
});
```

## Saga 패턴 (분산 트랜잭션)

여러 서비스에 걸친 트랜잭션을 각 서비스가 로컬 트랜잭션 + 보상 트랜잭션으로 처리.

### Choreography Saga (이벤트 기반)

```
주문 생성 흐름:
  OrderService  → order.created 이벤트 발행
  PaymentService → payment.completed 이벤트 발행
  InventoryService → inventory.reserved 이벤트 발행
  OrderService  → order.confirmed 이벤트 발행

실패 시 보상:
  inventory.reserve.failed → payment.refund 이벤트 발행
  payment.refund.completed → order.cancelled 이벤트 발행
```

```typescript
// OrderService — Choreography Saga
consumer.subscribe(['payment.completed', 'payment.failed']);

consumer.on('payment.completed', async (event) => {
  await inventoryService.reserve(event.orderId, event.items);
  await publishEvent('order.payment-confirmed', { orderId: event.orderId });
});

consumer.on('payment.failed', async (event) => {
  await orderRepository.updateStatus(event.orderId, 'CANCELLED');
  await publishEvent('order.cancelled', { orderId: event.orderId, reason: 'payment_failed' });
});
```

### Orchestration Saga (중앙 조율)

```typescript
// OrderSaga.ts — 중앙 조율자
class OrderSaga {
  async execute(orderId: string): Promise<void> {
    try {
      // 1. 결제
      await this.step('payment', () => paymentService.charge(orderId));

      // 2. 재고 예약
      await this.step('inventory', () => inventoryService.reserve(orderId));

      // 3. 배송 준비
      await this.step('shipping', () => shippingService.prepare(orderId));

      await orderRepository.updateStatus(orderId, 'CONFIRMED');

    } catch (err) {
      await this.compensate(orderId);
      throw err;
    }
  }

  private async compensate(orderId: string): Promise<void> {
    const completedSteps = await sagaRepository.getCompletedSteps(orderId);

    // 역순으로 보상 트랜잭션 실행
    for (const step of completedSteps.reverse()) {
      try {
        await this.rollback(step, orderId);
      } catch (err) {
        logger.error(`Compensation failed for step ${step}`, { orderId });
      }
    }

    await orderRepository.updateStatus(orderId, 'CANCELLED');
  }
}
```

## 통신 패턴 선택 기준

| 상황 | 권장 패턴 |
|------|------|
| 즉각 응답 필요 (조회) | REST 또는 gRPC |
| 내부 서비스 간 고성능 호출 | gRPC |
| 비즈니스 이벤트 전파 | Kafka 이벤트 발행 |
| 분산 트랜잭션 (단순) | Choreography Saga |
| 분산 트랜잭션 (복잡) | Orchestration Saga |
| 하위 서비스 장애 격리 | Circuit Breaker |
