# Node.js 작업 큐 처리

## 개요

HTTP 요청 안에서 처리하기엔 너무 오래 걸리거나, 실패 시 재시도가 필요하거나, 비동기로 처리해도 되는 작업은 큐로 분리한다.

```
큐로 분리해야 하는 작업:
  - 이메일/알림 발송 (외부 서비스, 실패 가능성)
  - 이미지 리사이징, 파일 변환
  - 대용량 데이터 집계/리포트 생성
  - 외부 API 연동 (결제, 배송 상태 조회)
  - 검색 인덱스 갱신
```

### 작업 등록 (Producer)

```typescript
// services/NotificationService.ts
import { emailQueue } from '../queues';

interface EmailJob {
  to: string;
  subject: string;
  template: string;
  data: Record<string, unknown>;
}

export class NotificationService {
  async sendEmail(payload: EmailJob): Promise<void> {
    await emailQueue.add('send-email', payload, {
      attempts: 3,                     // 실패 시 최대 3회 재시도
      backoff: {
        type: 'exponential',
        delay: 1000,                   // 1s, 2s, 4s 간격으로 재시도
      },
      removeOnComplete: { count: 100 }, // 완료 작업 100개만 보관
      removeOnFail: { count: 50 },      // 실패 작업 50개만 보관
    });
  }

  async sendEmailAt(payload: EmailJob, sendAt: Date): Promise<void> {
    const delay = sendAt.getTime() - Date.now();
    await emailQueue.add('send-email', payload, { delay });
  }

  async sendEmailBulk(payloads: EmailJob[]): Promise<void> {
    const jobs = payloads.map(payload => ({
      name: 'send-email',
      data: payload,
      opts: { attempts: 3 },
    }));
    await emailQueue.addBulk(jobs);
  }
}
```

### 작업 처리 (Worker)

```typescript
// workers/emailWorker.ts
import { Worker, Job } from 'bullmq';
import IORedis from 'ioredis';
import { sendEmail } from '../utils/mailer';
import { logger } from '../utils/logger';

const connection = new IORedis({
  host: process.env.REDIS_HOST ?? 'localhost',
  maxRetriesPerRequest: null,
});

const worker = new Worker(
  'email',
  async (job: Job) => {
    logger.info(`이메일 발송 시작`, { jobId: job.id, to: job.data.to });

    await sendEmail({
      to: job.data.to,
      subject: job.data.subject,
      template: job.data.template,
      data: job.data.data,
    });

    logger.info(`이메일 발송 완료`, { jobId: job.id });
    return { sentAt: new Date().toISOString() };
  },
  {
    connection,
    concurrency: 5,        // 동시에 5개 처리
    limiter: {
      max: 100,            // 분당 최대 100개
      duration: 60_000,
    },
  }
);

worker.on('completed', (job) => {
  logger.info('Job completed', { jobId: job.id });
});

worker.on('failed', (job, err) => {
  logger.error('Job failed', { jobId: job?.id, error: err.message });
});

worker.on('error', (err) => {
  logger.error('Worker error', { error: err.message });
});
```

## 진행 상황 추적

```typescript
// Worker에서 진행률 업데이트
const worker = new Worker('report', async (job: Job) => {
  const items = await fetchItems();

  for (let i = 0; i < items.length; i++) {
    await processItem(items[i]);
    await job.updateProgress(Math.round((i + 1) / items.length * 100));
  }

  return { totalProcessed: items.length };
});

// Producer에서 진행률 조회
const job = await reportQueue.getJob(jobId);
const progress = await job?.progress; // 0~100
const state = await job?.getState();  // waiting | active | completed | failed
```

## 작업 큐 모니터링 (Bull Board)

```typescript
// app.ts
import { createBullBoard } from '@bull-board/api';
import { BullMQAdapter } from '@bull-board/api/bullMQAdapter';
import { ExpressAdapter } from '@bull-board/express';
import { emailQueue, imageQueue, reportQueue } from './queues';

const serverAdapter = new ExpressAdapter();
serverAdapter.setBasePath('/admin/queues');

createBullBoard({
  queues: [
    new BullMQAdapter(emailQueue),
    new BullMQAdapter(imageQueue),
    new BullMQAdapter(reportQueue),
  ],
  serverAdapter,
});

app.use('/admin/queues', serverAdapter.getRouter());
// http://localhost:3000/admin/queues 에서 UI 확인
```

## 에러 처리 및 데드 레터

```typescript
// 최대 재시도 후 실패한 작업 처리
const queueEvents = new QueueEvents('email', { connection });

queueEvents.on('failed', async ({ jobId, failedReason }) => {
  const job = await emailQueue.getJob(jobId);
  if (!job) return;

  // 최대 재시도 횟수 초과 시 알림
  if (job.attemptsMade >= (job.opts.attempts ?? 1)) {
    await alertService.notify({
      type: 'job_permanently_failed',
      jobId,
      queue: 'email',
      data: job.data,
      reason: failedReason,
    });
  }
});
```

## 체크리스트

- [ ] Redis 연결에 `maxRetriesPerRequest: null` 설정
- [ ] Worker에 `concurrency` 적절히 설정 (기본 1)
- [ ] 모든 Job에 `attempts` + `backoff` 설정
- [ ] `removeOnComplete` / `removeOnFail` 설정 (메모리 관리)
- [ ] Worker 이벤트(`completed`, `failed`, `error`) 핸들링
- [ ] 장기 작업은 `job.updateProgress()` 활용
- [ ] Bull Board UI 또는 대체 모니터링 도구 구성
- [ ] Graceful shutdown 구현 (`worker.close()`)
