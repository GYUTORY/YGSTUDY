# Node.js 에러 핸들링 전략

## 개요

에러를 제대로 처리하지 않으면 서버가 예고 없이 죽거나, 클라이언트에 500 에러만 반환하거나, 에러 원인을 추적할 수 없다. Node.js는 비동기 중심 환경이라 에러 전파 방식이 다양하고 놓치기 쉽다.

```
흔한 실수:
  - async 함수에서 try/catch 없이 await
  - Promise reject를 처리하지 않음 (UnhandledPromiseRejection)
  - 에러 로그 없이 catch 블록에서 조용히 무시
  - 모든 에러에 500 응답 → 클라이언트가 원인을 모름
```

---

## 핵심 개념

### 에러 종류

| 종류 | 설명 | 예시 |
|------|------|------|
| **Operational Error** | 예측 가능한 런타임 에러 | DB 연결 실패, 유효성 검증 실패, 404 |
| **Programmer Error** | 코드 버그 | null 접근, 잘못된 타입, 무한 루프 |
| **System Error** | 환경/OS 에러 | ENOENT, ECONNREFUSED, ETIMEDOUT |

Operational Error는 잡아서 적절한 응답을 내려야 하고, Programmer Error는 서버를 재시작하는 것이 안전하다.

## 기본 에러 클래스 설계

```typescript
// errors/AppError.ts
export class AppError extends Error {
  constructor(
    public readonly message: string,
    public readonly statusCode: number,
    public readonly code: string,
    public readonly isOperational: boolean = true
  ) {
    super(message);
    this.name = this.constructor.name;
    Error.captureStackTrace(this, this.constructor);
  }
}

export class NotFoundError extends AppError {
  constructor(resource: string) {
    super(`${resource}을(를) 찾을 수 없습니다.`, 404, 'NOT_FOUND');
  }
}

export class ValidationError extends AppError {
  constructor(message: string) {
    super(message, 400, 'VALIDATION_ERROR');
  }
}

export class UnauthorizedError extends AppError {
  constructor(message = '인증이 필요합니다.') {
    super(message, 401, 'UNAUTHORIZED');
  }
}

export class ForbiddenError extends AppError {
  constructor(message = '권한이 없습니다.') {
    super(message, 403, 'FORBIDDEN');
  }
}

export class ConflictError extends AppError {
  constructor(message: string) {
    super(message, 409, 'CONFLICT');
  }
}
```

## Express 에러 핸들링

### 전역 에러 미들웨어

```typescript
// middleware/errorHandler.ts
import { Request, Response, NextFunction } from 'express';
import { AppError } from '../errors/AppError';
import { logger } from '../utils/logger';

export function errorHandler(
  err: Error,
  req: Request,
  res: Response,
  next: NextFunction
): void {
  if (err instanceof AppError && err.isOperational) {
    // 예측 가능한 에러 — 클라이언트에 상세 정보 전달
    res.status(err.statusCode).json({
      success: false,
      code: err.code,
      message: err.message,
    });
    return;
  }

  // Programmer Error — 로그 남기고 500 반환
  logger.error('Unexpected error', {
    message: err.message,
    stack: err.stack,
    url: req.url,
    method: req.method,
    body: req.body,
  });

  res.status(500).json({
    success: false,
    code: 'INTERNAL_SERVER_ERROR',
    message: '서버 내부 오류가 발생했습니다.',
  });
}
```

### async 라우터 래퍼

```typescript
// utils/asyncWrapper.ts
import { Request, Response, NextFunction, RequestHandler } from 'express';

type AsyncHandler = (req: Request, res: Response, next: NextFunction) => Promise<any>;

export const asyncWrapper = (fn: AsyncHandler): RequestHandler =>
  (req, res, next) => {
    fn(req, res, next).catch(next); // 에러를 next()로 전달
  };

// 사용 예시
router.get('/users/:id', asyncWrapper(async (req, res) => {
  const user = await userService.findById(req.params.id);
  if (!user) throw new NotFoundError('사용자');
  res.json(user);
}));
```

### app.ts 설정

```typescript
import express from 'express';
import { errorHandler } from './middleware/errorHandler';
import { notFoundHandler } from './middleware/notFoundHandler';

const app = express();

// 라우터 등록
app.use('/api', router);

// 404 핸들러 (라우터 등록 후)
app.use(notFoundHandler);

// 에러 핸들러 (항상 마지막)
app.use(errorHandler);
```

## NestJS 에러 핸들링

### 기본 제공 Exception

```typescript
import {
  NotFoundException,
  BadRequestException,
  UnauthorizedException,
  ForbiddenException,
  ConflictException,
  InternalServerErrorException,
} from '@nestjs/common';

@Injectable()
export class UserService {
  async findOne(id: string): Promise<User> {
    const user = await this.userRepository.findOne({ where: { id } });
    if (!user) {
      throw new NotFoundException(`사용자 ${id}를 찾을 수 없습니다.`);
    }
    return user;
  }
}
```

### 커스텀 Exception Filter

```typescript
// filters/http-exception.filter.ts
import {
  ExceptionFilter,
  Catch,
  ArgumentsHost,
  HttpException,
  HttpStatus,
  Logger,
} from '@nestjs/common';

@Catch()
export class GlobalExceptionFilter implements ExceptionFilter {
  private readonly logger = new Logger(GlobalExceptionFilter.name);

  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const request = ctx.getRequest();

    let statusCode = HttpStatus.INTERNAL_SERVER_ERROR;
    let message = '서버 내부 오류';
    let code = 'INTERNAL_SERVER_ERROR';

    if (exception instanceof HttpException) {
      statusCode = exception.getStatus();
      const exceptionResponse = exception.getResponse();
      message =
        typeof exceptionResponse === 'object'
          ? (exceptionResponse as any).message
          : exceptionResponse;
      code = exception.name.replace('Exception', '').toUpperCase();
    } else {
      this.logger.error('Unexpected error', exception instanceof Error ? exception.stack : exception);
    }

    response.status(statusCode).json({
      success: false,
      code,
      message,
      timestamp: new Date().toISOString(),
      path: request.url,
    });
  }
}
```

## 비동기 에러 처리 패턴

### Promise 체인

```typescript
// ❌ 에러 누락
async function bad() {
  const result = await doSomething(); // 에러 발생 시 UnhandledPromiseRejection
  return result;
}

// ✅ 명시적 처리
async function good() {
  try {
    const result = await doSomething();
    return result;
  } catch (err) {
    if (err instanceof KnownError) {
      throw new AppError(err.message, 400, 'KNOWN_ERROR');
    }
    throw err; // 모르는 에러는 그대로 위로 전파
  }
}
```

### Result 패턴 (에러를 값으로)

```typescript
type Result<T, E = Error> =
  | { ok: true; data: T }
  | { ok: false; error: E };

async function safeDbCall<T>(fn: () => Promise<T>): Promise<Result<T>> {
  try {
    const data = await fn();
    return { ok: true, data };
  } catch (error) {
    return { ok: false, error: error as Error };
  }
}

// 사용
const result = await safeDbCall(() => userRepo.findOne(id));
if (!result.ok) {
  throw new AppError('DB 조회 실패', 500, 'DB_ERROR');
}
const user = result.data;
```

## UnhandledRejection / UncaughtException

```typescript
// 프로세스 레벨 에러 처리 (최후 방어선)
process.on('uncaughtException', (err: Error) => {
  logger.error('UncaughtException — 프로세스 종료', { error: err.message, stack: err.stack });
  // Programmer Error이므로 재시작이 안전
  process.exit(1);
});

process.on('unhandledRejection', (reason: unknown) => {
  logger.error('UnhandledRejection', { reason });
  // PM2나 Kubernetes가 자동 재시작
  process.exit(1);
});
```

## 에러 응답 형식 표준화

```typescript
// 통일된 에러 응답 형식
interface ErrorResponse {
  success: false;
  code: string;       // 에러 코드 (프론트엔드에서 분기 처리용)
  message: string;    // 사람이 읽을 수 있는 메시지
  details?: unknown;  // 유효성 검증 에러 상세 (선택)
}

// 성공 응답과 대비
interface SuccessResponse<T> {
  success: true;
  data: T;
}
```
