# AWS S3 (Simple Storage Service) 💾

## S3란 무엇인가?

Amazon S3는 인터넷을 위한 객체 스토리지 서비스입니다. 쉽게 말해서, 인터넷에 연결된 거대한 하드디스크라고 생각하면 됩니다. 전통적인 파일 시스템과는 다르게, S3는 객체 기반 스토리지를 사용합니다.

### 객체 스토리지의 핵심 개념

**객체(Object)**
- S3에 저장되는 파일의 기본 단위
- 파일 자체와 메타데이터(파일 정보)로 구성
- 최소 0바이트부터 최대 5TB까지 저장 가능
- 각 객체는 고유한 키(Key)를 가짐

**버킷(Bucket)**
- 객체들을 담는 컨테이너 역할
- 전 세계에서 유일한 이름을 가져야 함
- 리전(지역)을 선택하여 생성
- DNS 호환 이름 규칙을 따라야 함

**리전(Region)과 가용 영역**
- 리전: AWS가 서비스를 제공하는 지리적 위치
- 가용 영역: 각 리전 내의 격리된 물리적 위치
- 데이터는 여러 가용 영역에 자동으로 복제되어 안정성 확보

## 스토리지 클래스별 특성과 활용

### S3 Standard
**특징**
- 가장 높은 가용성과 성능 제공
- 자주 접근하는 데이터에 최적화
- 99.99% 가용성, 99.999999999% 내구성

**활용 사례**
- 웹사이트 이미지, 동영상, 문서
- 실시간 데이터 처리
- 콘텐츠 배포

### S3 Intelligent-Tiering
**특징**
- 접근 패턴을 자동으로 분석하여 최적의 스토리지 티어 선택
- 모니터링 비용 없음
- 자동으로 비용 최적화

**활용 사례**
- 접근 패턴이 예측하기 어려운 데이터
- 사용자 업로드 파일
- 로그 데이터

### S3 Standard-IA (Infrequent Access)
**특징**
- Standard보다 약 40% 저렴
- 최소 30일 보관 필요
- 자주 접근하지 않는 데이터에 적합

**활용 사례**
- 백업 파일
- 재해 복구 데이터
- 장기 보관이 필요한 문서

### S3 One Zone-IA
**특징**
- 단일 가용 영역에만 저장
- Standard-IA보다 약 20% 저렴
- 재생성 가능한 데이터에 적합

**활용 사례**
- 보조 백업
- 재생성 가능한 데이터
- 비용 최적화가 중요한 경우

### S3 Glacier
**특징**
- 장기 보관에 최적화
- 검색 시간: 분~시간 단위
- 매우 저렴한 비용

**활용 사례**
- 규제 준수 데이터
- 장기 백업
- 아카이브 데이터

### S3 Glacier Deep Archive
**특징**
- 가장 저렴한 옵션
- 검색 시간: 12시간
- 7년 이상 보관에 적합

**활용 사례**
- 법적 보관 의무 데이터
- 장기 아카이브
- 비용이 최우선인 경우

## 보안과 접근 제어

### 암호화 옵션

**서버 측 암호화 (SSE)**
- SSE-S3: AWS가 관리하는 키로 암호화
- SSE-KMS: AWS KMS로 관리하는 키로 암호화
- SSE-C: 고객이 제공하는 키로 암호화

### 접근 제어 메커니즘

**IAM 정책**
- 사용자별 세밀한 접근 권한 관리
- 역할 기반 접근 제어
- 최소 권한 원칙 적용

**버킷 정책**
- 버킷 레벨의 접근 제어
- 조건부 접근 규칙 설정
- IP 주소, 시간, 사용자 기반 제어

**ACL (Access Control List)**
- 객체별 세밀한 접근 제어
- 퍼블릭 읽기/쓰기 권한 설정
- 특정 사용자나 그룹에 대한 권한 부여

**VPC 엔드포인트**
- 프라이빗 네트워크에서만 접근 가능
- 인터넷을 통한 트래픽 차단
- 보안 강화

**MFA 삭제**
- 다중 인증을 통한 삭제 보호
- 실수로 인한 데이터 손실 방지
- 중요한 데이터 보호

## 데이터 관리 기능

### 수명 주기 정책
파일의 나이와 접근 패턴에 따라 자동으로 다른 스토리지 클래스로 이동하는 기능입니다. 이를 통해 비용을 최적화하면서도 데이터 접근성을 유지할 수 있습니다.

**활용 예시**
- 30일 후: Standard → Standard-IA
- 90일 후: Standard-IA → Glacier
- 365일 후: Glacier → Glacier Deep Archive

### 버전 관리
파일의 모든 버전을 유지하여 데이터 보호를 강화합니다.

**주요 기능**
- 실수로 삭제된 파일 복구 가능
- 이전 버전으로 되돌리기 가능
- 변경 이력 추적
- 의도하지 않은 수정 방지

### 크로스 리전 복제
다른 리전에 자동으로 데이터를 복제하는 기능입니다.

**활용 목적**
- 지연 시간 최소화
- 규제 준수
- 재해 복구
- 글로벌 서비스 제공

## 주요 활용 사례

### 데이터 레이크
대규모 데이터를 저장하고 분석하는 중앙 저장소로 활용됩니다. 구조화된 데이터와 비구조화된 데이터를 모두 저장할 수 있어 빅데이터 분석에 적합합니다.

### 백업 및 복구
- 데이터 백업 자동화
- 재해 복구 솔루션
- 크로스 리전 복제를 통한 안정성 확보

### 정적 웹사이트 호스팅
HTML, CSS, JavaScript 파일을 직접 호스팅할 수 있습니다. CloudFront와 함께 사용하면 전 세계에 빠른 콘텐츠 배포가 가능합니다.

### 미디어 스토리지
이미지, 비디오, 오디오 파일을 저장하고 처리하는 데 활용됩니다. Lambda와 함께 사용하면 이미지 리사이징, 포맷 변환 등의 작업을 자동화할 수 있습니다.

### 로그 저장
서버 로그, 애플리케이션 로그, 감사 로그를 체계적으로 저장하고 관리할 수 있습니다. CloudWatch Logs와 연동하여 모니터링과 분석이 가능합니다.

## 모범 사례

### 버킷 명명 규칙
- 전역적으로 고유한 이름 사용
- DNS 호환 이름 (소문자, 숫자, 하이픈만)
- 3-63자 길이 유지
- 의미 있는 이름 사용

### 보안 모범 사례
- 최소 권한 원칙 적용
- 암호화 활성화
- 정기적인 보안 감사
- 접근 로그 모니터링

### 비용 최적화
- 적절한 스토리지 클래스 선택
- 수명 주기 정책 설정
- 불필요한 데이터 정리
- 요청 패턴 분석

### 성능 최적화
- 멀티파트 업로드 사용 (대용량 파일)
- 적절한 키 이름 구조 사용
- CloudFront와 함께 사용
- 적절한 청크 크기 설정

## 제한 사항과 비용

### 주요 제한 사항
| 항목 | 제한 |
|------|------|
| 단일 객체 크기 | 최대 5TB |
| 버킷당 객체 수 | 무제한 |
| 버킷 이름 길이 | 3-63자 |
| 버킷 정책 크기 | 20KB |
| 초당 요청 제한 | 3,500 PUT/COPY/POST/DELETE<br>5,500 GET/HEAD |

### 비용 구조 (us-east-1 기준)

**스토리지 비용**
- Standard: $0.023/GB/월
- Standard-IA: $0.0125/GB/월
- One Zone-IA: $0.01/GB/월
- Glacier: $0.004/GB/월
- Glacier Deep Archive: $0.00099/GB/월

**요청 비용**
- GET 요청: $0.0004/1,000건
- PUT 요청: $0.005/1,000건
- DELETE 요청: 무료

**데이터 전송 비용**
- 인바운드: 무료
- 아웃바운드: $0.09/GB (첫 1TB)

## S3의 안정성

### 내구성(Durability)
- **99.999999999% (11개의 9)** 내구성
- 1,000,000개의 파일 중 1개만 손실될 확률
- 연간 0.000000001%의 데이터 손실 가능성

### 가용성(Availability)
- **99.99%** 가용성
- 월간 다운타임이 52.56분 이하
- 여러 가용 영역에 자동 복제

## 마무리

AWS S3는 확장 가능하고 안전한 객체 스토리지 서비스입니다. 적절한 설정과 모범 사례를 따르면 비용 효율적이고 안전한 데이터 스토리지 솔루션을 구축할 수 있습니다. 

S3를 처음 사용하는 경우, AWS S3 콘솔에서 제공하는 "Getting Started" 가이드를 참고하시기 바랍니다. 또한 각 스토리지 클래스의 특성을 잘 이해하고 데이터의 특성에 맞는 클래스를 선택하는 것이 중요합니다.
