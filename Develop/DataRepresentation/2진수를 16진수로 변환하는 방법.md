# 2진수를 16진수로 변환하는 방법

## 왜 2진수와 16진수를 알아야 할까?

컴퓨터는 모든 정보를 0과 1로만 처리합니다. 하지만 사람이 2진수를 직접 다루기에는 너무 길고 복잡해요. 예를 들어 255라는 숫자를 2진수로 표현하면 11111111이 되는데, 이렇게 긴 숫자들을 매번 읽고 쓰기엔 부담스럽죠.

16진수는 이런 문제를 해결해줍니다. 2진수 4자리를 16진수 1자리로 압축할 수 있어서 훨씬 간결하고 읽기 쉬워요. 그래서 프로그래머들이 메모리 주소, 색상 코드, 파일 형식 등을 표현할 때 자주 사용합니다.

### 각 진수 시스템의 특징
- **2진수**: 컴퓨터가 이해하는 언어 (0, 1)
- **8진수**: Unix 시스템의 파일 권한 등에서 사용 (0~7)
- **10진수**: 우리가 일상에서 사용하는 숫자 (0~9)
- **16진수**: 프로그래밍에서 가장 많이 쓰이는 진수 (0~9, A~F)

## 변환의 핵심 원리

### 4자리씩 묶는 이유
16진수는 16개의 서로 다른 숫자를 사용합니다 (0~9, A~F). 2진수 4자리로 만들 수 있는 조합은 정확히 16가지입니다 (2⁴ = 16). 이게 바로 2진수 4자리를 16진수 1자리로 변환할 수 있는 이유예요.

### 변환 과정
1. 2진수를 오른쪽부터 4자리씩 묶어요
2. 각 묶음을 16진수로 바꿔요
3. 결과를 순서대로 연결해요

### 기본 변환표 (외워두면 편해요)
| 2진수 | 16진수 | 2진수 | 16진수 |
|-------|--------|-------|--------|
| 0000  | 0      | 1000  | 8      |
| 0001  | 1      | 1001  | 9      |
| 0010  | 2      | 1010  | A      |
| 0011  | 3      | 1011  | B      |
| 0100  | 4      | 1100  | C      |
| 0101  | 5      | 1101  | D      |
| 0110  | 6      | 1110  | E      |
| 0111  | 7      | 1111  | F      |

### 자릿수가 안 맞을 때
2진수의 자릿수가 4의 배수가 아니면, 앞에 0을 붙여서 맞춰줍니다. 예를 들어 1011은 00001011로 만들어서 변환해요.

### 실제 변환 예시

**예시 1: 1101011011**
1. 4자리씩 묶기: 11 | 0101 | 1011
2. 각각 변환: 3 | 5 | B
3. 결과: 0x35B

**예시 2: 1011 (자릿수 부족)**
1. 앞에 0 추가: 00001011
2. 4자리씩 묶기: 0000 | 1011
3. 각각 변환: 0 | B
4. 결과: 0x0B

## 수동으로 변환하는 방법

### 단계별 과정

**1단계: 자릿수 맞추기**
2진수의 길이가 4의 배수가 아니면 앞에 0을 붙여서 맞춰요.
- 1011 → 00001011
- 110101 → 00110101

**2단계: 4자리씩 묶기**
오른쪽부터 4자리씩 나눠요.
- 00001011 → 0000 | 1011
- 00110101 → 0011 | 0101

**3단계: 각 묶음을 16진수로 변환**
위의 변환표를 참고해서 바꿔요.
- 0000 → 0
- 1011 → B
- 0011 → 3
- 0101 → 5

**4단계: 결과 연결**
앞에 0x를 붙이고 연결해요.
- 0x0B
- 0x35

## 프로그래밍에서 변환하기

### JavaScript
```javascript
// 가장 간단한 방법
const binary = "1101011011";
const decimal = parseInt(binary, 2);
const hex = "0x" + decimal.toString(16).toUpperCase();
console.log(hex); // "0x35B"
```

### Python
```python
# Python에서 변환
binary = "1101011011"
decimal = int(binary, 2)
hex_value = hex(decimal)
print(hex_value)  # "0x35b"
```

### Java
```java
// Java에서 변환
String binary = "1101011011";
int decimal = Integer.parseInt(binary, 2);
String hex = "0x" + Integer.toHexString(decimal).toUpperCase();
System.out.println(hex); // "0x35B"
```

## 실제로 어디서 쓰일까?

### 메모리 주소
컴퓨터의 메모리 주소는 보통 16진수로 표시해요. 예를 들어 0x7FFF1234 같은 형태로요. 이렇게 하면 주소를 더 간결하게 표현할 수 있어요.

### 색상 코드
웹에서 색상을 표현할 때 #FF0000 (빨간색), #00FF00 (초록색) 같은 형태로 쓰죠. 이게 바로 16진수예요. 각각 Red, Green, Blue 값을 16진수로 표현한 거예요.

### 파일 형식
많은 파일 형식에서 헤더 정보를 16진수로 저장해요. 예를 들어 JPEG 파일은 FF D8 FF로 시작하고, PNG 파일은 89 50 4E 47로 시작해요.

### 네트워크
MAC 주소나 IP 주소의 일부를 16진수로 표현하기도 해요. 00:1A:2B:3C:4D:5E 같은 형태가 바로 그것이죠.

## 주의할 점들

### 큰 수 처리
일반적인 정수 범위를 넘어서는 큰 2진수를 다룰 때는 BigInt나 BigInteger 같은 특별한 자료형을 사용해야 해요. JavaScript의 parseInt는 53비트까지만 정확하게 처리할 수 있어요.

### 음수 처리
2진수에서 음수를 표현하는 방법(2의 보수)이 있어요. 단순히 변환표만 사용하면 음수는 제대로 변환되지 않을 수 있어요.

### 유효성 검사
2진수 문자열에 0과 1 이외의 문자가 들어있으면 오류가 발생해요. 변환하기 전에 입력값을 검증하는 것이 좋아요.

## 실무에서 유용한 팁

### 자주 쓰는 값들 외워두기
- 1111 = F (15)
- 1110 = E (14)
- 1100 = C (12)
- 1000 = 8 (8)

이런 값들은 자주 나와서 외워두면 변환이 빨라져요.

### 온라인 도구 활용
간단한 변환이 필요할 때는 온라인 변환기를 사용하는 것도 좋아요. 하지만 원리를 이해하고 있는 상태에서 사용해야 해요.

### 디버깅할 때
프로그램에서 오류가 발생했을 때, 메모리 덤프나 로그를 보면 16진수로 된 값들이 많이 나와요. 이런 값들을 2진수로 변환해서 비트 단위로 분석해야 할 때가 있어요.

## 마무리

2진수와 16진수 변환은 프로그래밍의 기초 중 하나예요. 처음엔 복잡해 보일 수 있지만, 4자리씩 묶는 원리만 이해하면 금방 익숙해질 거예요.

특히 메모리 주소, 색상 코드, 파일 형식 등을 다룰 때 자주 사용되니까, 기본적인 변환표는 외워두는 게 좋아요. 그리고 프로그래밍할 때는 언어의 내장 함수를 활용하는 것이 가장 효율적이에요.

### 진수 시스템 정리
| 진수 | 기수 | 사용 문자 | 주로 쓰이는 곳 |
|------|------|-----------|----------------|
| 2진수 | 2 | 0, 1 | 컴퓨터 내부 |
| 8진수 | 8 | 0~7 | Unix 권한 |
| 10진수 | 10 | 0~9 | 일상생활 |
| 16진수 | 16 | 0~9, A~F | 프로그래밍 |
